Recordings from Previous Enablement Group

The recordings folder captures prior live enablement sessions: walkthroughs, Q&A, and small workshops. The fastest way to internalize the parts of the platform that are easier to watch than to read — Block Kit composition and OAuth flow especially.

Some platform concepts are hard to fully understand from docs alone — Block Kit composition, OAuth flow, modal state shape, debugging a failed signature. Watching a teammate walk through them with a working app on screen lands the concepts ten times faster.

Tip: watch at 1.25–1.5×. Pause on any payload JSON you don't recognize and cross-reference with the Slack API method docs. Take screenshots of useful snippets.

Suggested watch order

  1. The "intro to Slack platform" / orientation session (anchors all other sessions).
  2. OAuth + installation flow walkthrough — the part most devs misunderstand.
  3. Block Kit composition — see modals being built block-by-block in real time.
  4. Bolt for JS deep-dive — listener types, middleware, error handling.
  5. Q&A sessions — these usually contain the "why does X behave this way?" gold.

What to capture as you watch

  • Code snippets the speaker shows that aren't in the docs.
  • Workspace screenshots showing how UI renders in practice.
  • Common errors + fixes the speaker debugs live.
  • Timestamps for sections you want to revisit.

Key takeaways

  • Recordings teach you the unwritten conventions — watch 2 or 3 before your first PR.
  • Pause on payloads; cross-reference with API method docs.
  • Q&A clips are gold — that's where the "actually happens" answers live.
