TypeScript Basics

TypeScript is JavaScript with a static type system bolted on. The Programming with Mosh tutorial walks through the type system and tooling; this page focuses on the patterns you'll actually use inside a Bolt for JS Slack app.

Prereq forBolt SDK, Block Kit typing, refactor safety

The Slack JavaScript SDK ships first-class TypeScript definitions. That means in a TS Slack app, you get autocomplete on every Web API method, every payload field, every Block Kit element, and you catch shape mismatches at build time instead of inside a 3-second ack window.

Why bother: Slack payloads are deeply nested JSON. Without types, every field access is a guess. With types, your editor tells you that body.actions[0].selected_option is real and body.actions[0].selected_user is not.

The type system in one screen

Primitive types

let userId: string = 'U123';
let count: number = 5;
let isAdmin: boolean = false;
let payload: unknown = JSON.parse(raw); // safer than `any`

Object types & interfaces

interface InstallStore {
  workspaceId: string;
  botToken: string;
  installedAt: Date;
  scopes: string[];
}

const install: InstallStore = {
  workspaceId: 'T1',
  botToken: 'xoxb-...',
  installedAt: new Date(),
  scopes: ['chat:write', 'commands'],
};

Union & literal types

type SlackTokenPrefix = 'xoxb' | 'xoxp' | 'xapp';
type Status = 'pending' | 'approved' | 'rejected';

Generics

function first<T>(arr: T[]): T | undefined {
  return arr[0];
}
const channel = first(channels); // type inferred as Channel | undefined

Type narrowing & discriminated unions

Slack action types are a textbook discriminated union — the type field tells you which payload shape you have. 

type Action =
  | { type: 'button';      action_id: string; value: string }
  | { type: 'static_select'; selected_option: { value: string } }
  | { type: 'users_select';  selected_user: string };

function handle(a: Action) {
  if (a.type === 'button') return a.value;            // narrowed
  if (a.type === 'static_select') return a.selected_option.value;
  return a.selected_user;                              // exhaustively narrowed
}

tsconfig.json — the dial that matters

The compiler reads tsconfig.json. The one setting that changes your life is "strict": true. Turn it on; fix the errors. Modern Slack samples use it by default. 

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "strict": true,
    "esModuleInterop": true,
    "outDir": "dist",
    "rootDir": "src"
  }
}

Bolt + TypeScript example

import { App, BlockAction, ButtonAction } from '@slack/bolt';

const app = new App({
  token: process.env.SLACK_BOT_TOKEN!,
  signingSecret: process.env.SLACK_SIGNING_SECRET!,
});

app.action<BlockAction<ButtonAction>>('approve_button', async ({ ack, body, client }) => {
  await ack();
  await client.chat.postMessage({
    channel: body.channel!.id,
    text: `Approved by <@${body.user.id}>`,
  });
});

Notice how the generic on app.action tells TypeScript exactly what shape body is — including the typed actions array.

Common pitfalls (and fixes)

  • Reaching for any: use unknown + a type guard instead.
  • Using ! everywhere: non-null assertion bypasses the compiler. Validate or narrow instead.
  • Wide function signatures: a function that takes object tells you nothing. Define an interface.
  • Skipping strictNullChecks: the single biggest bug-prevention setting.

Key takeaways

  • Always start a new Slack app project in TypeScript with strict: true.
  • Lean on interfaces for installation stores, settings, and persisted models.
  • Use discriminated unions to model Slack actions and event types.
  • Generics on app.action<...> unlock typed payloads — use them.
