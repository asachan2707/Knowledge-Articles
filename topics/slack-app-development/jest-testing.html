JavaScript Testing with Jest

Jest is the de-facto JavaScript test framework. The freeCodeCamp crash course covers the runner, matchers, mocks, and async patterns. This page focuses on the patterns you'll actually use to unit-test Bolt for JS handlers — fake payloads, mocked Web API client, and asserting on chat.postMessage arguments.

GoalConfidently unit-test Slack handlers
Time~90 min crash course + practice
Prereq forRefactor safety, CI, on-call confidence

Without tests, every change to a Slack app handler is a small bet that no one will notice. With tests, you can refactor a slash command, add a new modal field, or change an API call and know the wiring is intact before you deploy.

Pragmatic target: 1 unit test per handler that confirms the right Web API method gets called with the right arguments, plus a few integration tests for the OAuth flow. You don't need 100% coverage; you need confidence on the hot paths.

Jest in 5 concepts

  • Test runner: describe, it / test, beforeEach, afterEach.
  • Matchers: expect(x).toBe(y), toEqual (deep), toContain, toThrow, resolves / rejects for promises.
  • Mocks: jest.fn() creates a callable mock; jest.mock('module') replaces an entire module.
  • Spies: jest.spyOn(obj, 'method') replaces one method while keeping the rest.
  • Snapshots: toMatchSnapshot() for stable structured output (Block Kit JSON).

Test a Bolt slash command (worked example)

handler under test

// src/commands/report.js
export function registerReport(app) {
  app.command('/report', async ({ command, ack, client }) => {
    await ack();
    await client.chat.postMessage({
      channel: command.channel_id,
      text: `Hello <@${command.user_id}>, here is your report.`,
    });
  });
}

jest test

// src/commands/report.test.js
import { registerReport } from './report.js';

describe('/report command', () => {
  it('acks and posts a message to the same channel', async () => {
    const ack = jest.fn();
    const postMessage = jest.fn();
    const client = { chat: { postMessage } };

    // capture the handler that registerReport registers
    let handler;
    const app = { command: (_name, h) => { handler = h; } };
    registerReport(app);

    await handler({
      command: { channel_id: 'C1', user_id: 'U1' },
      ack,
      client,
    });

    expect(ack).toHaveBeenCalledTimes(1);
    expect(postMessage).toHaveBeenCalledWith({
      channel: 'C1',
      text: 'Hello <@U1>, here is your report.',
    });
  });
});

Mocking the Slack Web API client

For broader integration tests, you can mock @slack/web-api at the module level. Every method becomes a jest.fn() that resolves to whatever you want. 

jest.mock('@slack/web-api', () => ({
  WebClient: jest.fn().mockImplementation(() => ({
    chat:           { postMessage: jest.fn().mockResolvedValue({ ok: true, ts: '123' }) },
    conversations:  { list:        jest.fn().mockResolvedValue({ ok: true, channels: [] }) },
    users:          { info:        jest.fn().mockResolvedValue({ ok: true, user: { id: 'U1' } }) },
  })),
}));

Snapshot-test your Block Kit

Block Kit JSON is stable structured output — perfect for snapshot tests. A snapshot tells you when a UI change was intentional vs. an accident. 

import { buildApprovalModal } from './blocks.js';

it('approval modal blocks', () => {
  const view = buildApprovalModal({ requesterId: 'U1', amount: 500 });
  expect(view.blocks).toMatchSnapshot();
});

Test pyramid for a Slack app

  • Unit (lots): handler logic with mocked client + ack — fast.
  • Integration (some): in-memory DB + mocked Web API; exercise OAuth flow.
  • End-to-end (few): point a sandbox Slack workspace at a deployed staging app and run a smoke test.

Common Jest gotchas

  • Forgetting to await async expectations — the test passes silently.
  • Module-level mocks must be hoisted; declare them at the top of the file.
  • jest.clearAllMocks() in beforeEach to avoid bleed-through.
  • For ESM projects, configure "transform" or use --experimental-vm-modules.

Key takeaways

  • Test handlers by capturing them at registration, then calling with fake payloads.
  • Mock @slack/web-api at the module level; assert on call args, not internals.
  • Snapshot Block Kit JSON to catch unintended UI drift.
  • Aim for fast unit tests on hot paths; one e2e smoke test against a sandbox workspace.
