Express.js Web Framework

Express is the minimalist HTTP framework most Node services are built on — including the HTTP receiver inside Bolt for JS. Master middleware, routing, error handling, and request validation here, and Bolt becomes "Express with Slack-shaped niceties on top."

Express is two ideas: routes map an HTTP method + URL pattern to a handler, and middleware are functions chained in order that can read/modify the request, short-circuit the response, or pass control to the next handler. That's the entire mental model.

Slack tie-in: Bolt's HttpReceiver is an Express app under the hood. When you mount a Bolt app at /slack/events alongside your own dashboard routes, you're using exactly the patterns this video teaches.

The middleware pattern in one example

import express from 'express';

const app = express();

// 1. Built-in middleware: parse JSON bodies
app.use(express.json());

// 2. Custom middleware: timing
app.use((req, res, next) => {
  const start = Date.now();
  res.on('finish', () => {
    console.log(`${req.method} ${req.url} - ${Date.now() - start}ms`);
  });
  next();
});

// 3. Route
app.get('/health', (req, res) => res.json({ ok: true }));

// 4. Error-handling middleware (4 args)
app.use((err, req, res, next) => {
  console.error(err);
  res.status(500).json({ error: 'internal' });
});

app.listen(3000);

Order matters. Middleware runs top-to-bottom; whichever one calls res.send()/res.json() wins. Error-handling middleware (the one with four arguments) only runs when something throws or calls next(err).

Routing patterns you'll use

  • app.get('/api/users/:id', handler) — route params via req.params.id.
  • app.post('/api/users', handler) — JSON body via req.body.
  • app.use('/admin', requireAuth, adminRouter) — group routes under a router.
  • app.all('*', notFoundHandler) — catch-all 404.

For organization, split routes into express.Router() modules and mount them. A Slack app commonly has /slack/events (Bolt), /slack/oauth/callback, /health, and /admin.

Body parsing — and why Slack is special

Slack's request signing requires the raw request body to verify HMAC. If you call express.json() before the Slack route, you'll lose the raw bytes and signature checks will fail. Bolt handles this internally if you let it own the route. If you mount your own Express around Bolt, register Bolt first and JSON parsing only on your other routes.

Error handling that doesn't lie

Three rules:

  1. Wrap async route handlers so thrown errors hit your error middleware (express-async-errors or asyncHandler wrapper).
  2. Never leak stack traces to clients — return a stable error code and log the detail server-side.
  3. Have one final error handler that returns JSON for API routes and HTML for HTML routes.
const asyncHandler = fn => (req, res, next) =>
  Promise.resolve(fn(req, res, next)).catch(next);

app.get('/users/:id', asyncHandler(async (req, res) => {
  const user = await db.users.findById(req.params.id);
  if (!user) return res.status(404).json({ error: 'not_found' });
  res.json(user);
}));

Security middleware checklist

  • helmet() — sets ~15 security headers.
  • cors() — explicit allowlist, never * in production.
  • express-rate-limit — protects against brute force and accidental loops.
  • compression() — gzip responses, free latency win.
  • Validate every input with zod or joi.

How Bolt sits on top

Bolt for JS exposes app.receiver.router (when using ExpressReceiver) so you can attach your own Express routes alongside Slack handlers. This is how you add an install URL, a privacy policy page, or an admin dashboard inside the same Node process as your Slack app. 

import { App, ExpressReceiver } from '@slack/bolt';

const receiver = new ExpressReceiver({ signingSecret: process.env.SLACK_SIGNING_SECRET });
const app = new App({ token: process.env.SLACK_BOT_TOKEN, receiver });

receiver.router.get('/health', (req, res) => res.send('ok'));
receiver.router.get('/install', (req, res) => res.redirect(await app.installer.generateInstallUrl(...)));

await app.start(3000);

Key takeaways

  • Express = routes + middleware in order. That's it.
  • Order matters: register Slack/Bolt first if you need raw body for signing.
  • Always wrap async handlers; always have a final JSON error handler.
  • Bolt's ExpressReceiver lets you mix Slack and your own HTTP routes.
