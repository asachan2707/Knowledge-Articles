ES6+ — Modern JavaScript

Programming with Mosh's "ES6 in 1 Hour" covers every feature added since 2015 — and almost every line of Bolt for JS uses at least one of them. This page maps each ES6+ feature to where it appears in real Slack code.

TrackSlack prereq · Tech Stack 1

Time~1 hour video + practice
Prereq forTypeScript, Bolt, every example app

ES6 (ECMAScript 2015) and the yearly releases since are the version of JavaScript everyone now writes. Slack's docs, examples, and the Bolt SDK lean on these features so heavily that the older syntax (var, function expressions, prototype chains) almost never shows up.

Mental model: ES6 turned JavaScript from a quirky scripting language into a modern app language — modules, classes, real iteration, real async, and ergonomic data shaping all landed at once.

The ES6+ feature map

let & const

Block-scoped, no hoisting weirdness. Use const by default.

Arrow functions

Compact syntax and lexical this. Bolt listeners are almost always arrow functions to avoid any this surprises inside callbacks. 

app.action('approve_button', async ({ ack, body, client }) => {
  await ack();
  await client.chat.postMessage({ channel: body.channel.id, text: 'Approved!' });
});

Template literals

Backticks + ${...} for strings with embedded expressions and multi-line strings — perfect for Slack message bodies.

Destructuring

Pull fields out of objects and arrays inline. Slack payloads are big — destructuring keeps handlers readable. 

// Slack action payload destructuring
const { user: { id: userId, name }, view: { state: { values } } } = body;

Default + rest + spread

Default parameters (function f(x = 0)), rest params (function f(...args)), and the spread operator ({ ...obj, foo: 1 }) are everywhere in modern JS. Spread is how you safely build new Block Kit blocks without mutating shared state.

Modules (ESM)

import/export replaced CommonJS require. Both still work in Node; new code mostly uses ESM. 

import { App } from '@slack/bolt';
import { db } from './db.js';
export const app = new App({ token: process.env.SLACK_BOT_TOKEN });

Classes

Sugar over prototypes. Useful for organizing services in a Slack app (class InstallationStore { ... }).

Promises & async/await

Promise is the type the Slack Web API client returns. async/await is how you consume it without callback hell. Always try/catch around await.

Iterators & generators

for-of, the iterable protocol, and function* generators. You'll mostly use for-of when paginating Slack API results.

Map / Set / WeakMap / WeakSet

Real collection types with stable iteration order and any-key support.

Newer (ES2017–ES2024) wins to know

  • Object.entries / Object.fromEntries for object↔array conversion.
  • Optional chaining ?. and nullish coalescing ??.
  • Promise.allSettled, Promise.any.
  • Numeric separators (1_000_000), top-level await.

Where this shows up in Bolt for JS

Every Bolt listener is an arrow function with destructured arguments. Almost every payload access uses optional chaining. Every Block Kit construction uses spread to compose blocks. Every Web API call returns a Promise consumed with await. ES6+ is the working idiom of the SDK.

Practice exercise

  1. Take a small ES5 codebase you wrote (or a snippet from the JS fundamentals video) and rewrite it using arrow functions, destructuring, spread, and async/await.
  2. Convert any require()/module.exports to import/export with "type": "module" in package.json.
  3. Try writing a small "fake Slack handler" that uses every ES6 feature in this list at least once.

Key takeaways

  • Arrow functions + destructuring are the default shape of Bolt listeners.
  • Spread is how you compose Block Kit blocks without mutation bugs.
  • Optional chaining (?.) is a survival skill against deeply nested Slack payloads.
  • async/await + try/catch is the only sane way to consume the Slack Web API.
