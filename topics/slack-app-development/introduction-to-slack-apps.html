Introduction to Slack Apps

Slack's "Start" guide explains the lifecycle of a Slack app: create it in the developer console, choose features, pick a hosting/transport path, and install it to a workspace. This page expands each step with the decisions you'll regret skipping later.

GoalCreate & install your first Slack app
Time~30 min read + 30 min hands-on
Prereq forAuthentication, Bolt quickstart

Every Slack app starts the same way: register at api.slack.com/apps, pick a name and a workspace, then enable the features you need (slash commands, event subscriptions, interactivity, OAuth, App Home). The decisions you make here — transport, scopes, distribution — shape the rest of the project.

Two big choices upfront: 1) Transport — HTTP receiver (public URL) vs. Socket Mode (no public URL needed). 2) Distribution — internal (one workspace, single bot token) vs. distributed (any workspace, full OAuth + installation store).

Step 1 — Create the app

  1. Go to api.slack.com/apps → "Create New App".
  2. Pick "From scratch" or "From an app manifest" (YAML/JSON). Manifests are reproducible.
  3. Choose your development workspace (a sandbox you own; never your prod workspace).

A minimal manifest

display_information:
  name: My Slack App
features:
  bot_user:
    display_name: my-app
  slash_commands:
    - command: /hello
      description: Say hi
      usage_hint: "[name]"
oauth_config:
  scopes:
    bot:
      - chat:write
      - commands
settings:
  event_subscriptions:
    bot_events:
      - app_mention
  interactivity:
    is_enabled: true
  org_deploy_enabled: false
  socket_mode_enabled: true

Step 2 — Choose a transport

HTTP receiver

Your app exposes an HTTPS URL (e.g., https://example.com/slack/events). Slack POSTs events and interactivity payloads to it. Requires a public URL — locally you'll use ngrok or Cloudflare Tunnel during development.

Socket Mode

Your app opens a WebSocket to Slack and receives events that way. No public URL required — ideal for local dev, behind-the-firewall apps, and demos. Not recommended for production distributed apps.

Step 3 — Pick features

  • Bot user — almost always yes; this is your app's identity.
  • Slash commands — global verbs your users invoke (/standup, /report).
  • Event subscriptions — react to messages, mentions, channel changes.
  • Interactivity & shortcuts — buttons, menus, modals, global & message shortcuts.
  • App Home — your app's own tab inside Slack (great for state & settings).
  • OAuth & permissions — declare scopes; mandatory for distribution.

Step 4 — Internal vs. distributed

An internal app only installs in your own workspace. You can copy the bot token from the install page and run with it. A distributed app supports installation by any workspace via OAuth — you must run an install URL endpoint, an OAuth redirect endpoint, and an installation store keyed by team/enterprise.

Going from internal to distributed later is doable but not free. Decide early; if there's any chance another team will use your app, make it distributed from day one.

Step 5 — Install & verify

  1. Install to your workspace from the app management page (or via your OAuth install URL for distributed apps).
  2. Copy the bot token (xoxb-…) and signing secret into your .env.
  3. Run your Bolt app and post /hello from the workspace — round-trip success is your "hello world".

Common first-app mistakes

  • Skipping the manifest — makes your app non-reproducible across environments.
  • Asking for too many scopes upfront — scares users and triggers re-consent.
  • Using your prod workspace as the dev workspace — recipe for embarrassing accidents.
  • Forgetting to enable interactivity when you want buttons / modals.
  • Using Socket Mode for production distributed apps.

Key takeaways

  • Always create the app with a manifest — reproducible across dev/staging/prod.
  • Pick transport based on environment: Socket Mode for dev, HTTP for prod.
  • Pick distribution based on intent: internal for one team, distributed for many.
  • Smallest scope set that works; add more only when forced.
