Bolt for JavaScript

Bolt for JS is Slack's official Node framework. It abstracts request signing, the 3-second ack, OAuth, the Web API client, and gives you ergonomic listeners. This page covers the framework's mental model, listener types, middleware, and the ship-ready patterns.

Bolt is one class — App — and a set of listeners. app.command(...) handles slash commands, app.action(...) handles block actions, app.view(...) handles modal submissions, app.event(...) handles Events API events, app.shortcut(...) handles global & message shortcuts. That's the entire surface.

The promise of Bolt: you only write what's unique to your app. Signing, ack timing, OAuth flow, retry logic, the Web API typing — all handled by the framework.

Hello world

import 'dotenv/config';
import { App } from '@slack/bolt';

const app = new App({
  token: process.env.SLACK_BOT_TOKEN,
  signingSecret: process.env.SLACK_SIGNING_SECRET,
  port: 3000,
});

app.command('/hello', async ({ command, ack, client }) => {
  await ack();
  await client.chat.postMessage({
    channel: command.channel_id,
    text: `Hi <@${command.user_id}>!`,
  });
});

await app.start();
console.log('⚡ Bolt app is running');

Listener types — full surface

// Slash command
app.command('/standup',          async ({ command, ack, client })           => { ... });

// Global shortcut (callback_id from your manifest)
app.shortcut('open_feedback',    async ({ shortcut, ack, client })          => { ... });

// Message shortcut (also via app.shortcut, type === 'message_action')
app.shortcut('save_message',     async ({ shortcut, ack, client })          => { ... });

// Block action — button click, select change
app.action('approve_button',     async ({ ack, body, client, action })      => { ... });

// Modal submission
app.view('feedback_modal',       async ({ ack, body, view, client })        => { ... });

// Events API event
app.event('app_mention',         async ({ event, client })                  => { ... });
app.message('hello',             async ({ message, say })                   => { say('Hi!'); });

// Options request (dynamic dropdowns)
app.options('approver_select',   async ({ ack, options })                   => { ... });

// Errors
app.error(async (error) => { console.error(error); });

The receiver

Bolt routes everything through a receiver. The defaults:

  • HTTPReceiver — built on Node's http; the default.
  • ExpressReceiver — uses Express so you can mount your own routes alongside Slack handlers.
  • SocketModeReceiver — opens a WebSocket; activated by passing socketMode: true.
  • AwsLambdaReceiver — for serverless deployments.

Middleware — the secret weapon

Bolt's middleware runs before each listener. You can add global middleware (app.use(fn)) that runs for every event, or scoped middleware on a single listener. 

// Global: log every incoming event
app.use(async ({ next, body }) => {
  console.log(`[in] ${body.type} ${body.api_app_id ?? ''}`);
  await next();
});

// Scoped: only run if the user is an admin
const adminOnly = async ({ context, body, next }) => {
  const userId = body.user.id ?? body.user_id;
  if (!isAdmin(userId)) return;  // do not call next() — short-circuit
  await next();
};

app.command('/admin', adminOnly, async ({ command, ack }) => { ... });

OAuth + InstallProvider for distributed apps

const app = new App({
  signingSecret: process.env.SLACK_SIGNING_SECRET,
  clientId:      process.env.SLACK_CLIENT_ID,
  clientSecret:  process.env.SLACK_CLIENT_SECRET,
  stateSecret:   process.env.SLACK_STATE_SECRET,
  scopes: ['chat:write', 'commands', 'app_mentions:read'],
  installationStore: {
    storeInstallation:  async (i)  => db.installations.upsert(i),
    fetchInstallation:  async (q)  => db.installations.findOne(q),
    deleteInstallation: async (q)  => db.installations.delete(q),
  },
  installerOptions: {
    directInstall: true,           // skip the consent landing page
    redirectUriPath: '/slack/oauth_redirect',
  },
});

Bolt automatically mounts /slack/install, /slack/oauth_redirect, and /slack/events. You write the installation store; Bolt calls it.

Project structure that scales

src/
  app.ts                 # boot Bolt + register modules
  install-store.ts       # InstallationStore impl
  middleware/
    log.ts
    require-admin.ts
  features/
    standup/
      command.ts         # app.command('/standup', ...)
      modal.ts           # views.open + view handlers
      blocks.ts          # builder functions returning blocks
      service.ts         # business logic (no Slack imports!)
    feedback/
      ...
  shared/
    db.ts
    redis.ts
    secrets.ts

The discipline: pure business logic in service.ts, Slack-aware glue in command.ts/modal.ts. This makes services unit-testable without Bolt.

Testing & observability

  • Unit-test handlers by capturing them at registration; mock ack + client.
  • Add request/response logging middleware in dev; structured logs in prod.
  • Send errors to Sentry/Datadog from app.error.
  • Deploy with at least 2 replicas behind a load balancer; Bolt is stateless if you push state to Mongo/Redis.

Key takeaways

  • One App + per-feature listener files = clean structure that scales.
  • Middleware short-circuits with "don't call next()"; use it for auth and gating.
  • For distributed apps, write a real installation store from day one.
  • Keep business logic free of Slack imports — testable without the framework.
