Circuit Breaker

Wraps calls to downstream services and monitors failure rates. When failures exceed a threshold, it "opens" — stopping all calls and returning a fallback immediately. Protects your system from cascading failures and gives struggling services time to recover.

Concept07 of 13
CategoryResilience Patterns
Pattern OriginMichael Nygard, "Release It!" (2007)
Used ByNetflix Hystrix, Resilience4j, AWS SDK

Without a circuit breaker, one slow or failed downstream service causes a cascade: your service threads block waiting for responses, your connection pool exhausts, your own service becomes slow, and now the service calling you also starts failing. In a microservices system, one degraded dependency can bring down the entire call chain. The circuit breaker stops this by failing fast.

Named after electrical circuit breakers: In your home, a circuit breaker trips when current exceeds a safe threshold — stopping damage to the wiring. In software, the circuit breaker "trips" when downstream error rate exceeds a threshold — stopping your system from making things worse for a struggling dependency.

State Machine

Three states: Closed, Open, Half-Open

CLOSED (normal operation) │ All calls pass through to downstream │ Failure counter increments on each failure │ │ failure rate > threshold (e.g. 50% in last 60s) ▼ OPEN (tripped — fast fail) │ All calls return immediately with fallback │ No calls reach downstream service (it gets a break) │ Downstream has time to recover │ │ after reset timeout (e.g. 30 seconds) ▼ HALF-OPEN (probe) │ Allow ONE test request through │ ├──▶ SUCCESS ──▶ reset counters ──▶ CLOSED (service recovered) └──▶ FAILURE ──▶ stay/return ──▶ OPEN (still broken, try again later)

The half-open state is critical. Without it, you'd need manual intervention to re-enable calls after recovery. The half-open state automates recovery detection: let one request through, if it succeeds the service is back, reset and resume normal traffic.

Implementation

Circuit breaker in Node.js — from scratch

const STATES = { CLOSED: 'CLOSED', OPEN: 'OPEN', HALF_OPEN: 'HALF_OPEN' };

class CircuitBreaker {
  constructor(options = {}) {
    this.failureThreshold  = options.failureThreshold  ?? 5;    // failures before opening
    this.failureRatePercent = options.failureRatePercent ?? 50;  // % failure rate threshold
    this.resetTimeout      = options.resetTimeout      ?? 30000; // ms before half-open probe
    this.halfOpenMaxCalls  = options.halfOpenMaxCalls  ?? 1;     // probes in half-open

    this.state          = STATES.CLOSED;
    this.failureCount   = 0;
    this.successCount   = 0;
    this.lastFailureTime = null;
    this.halfOpenCalls  = 0;

    // Sliding window: track last N call results
    this.callWindow     = [];
    this.windowSize     = options.windowSize ?? 20;
  }

  async call(fn, fallback) {
    if (this.state === STATES.OPEN) {
      // Check if reset timeout has elapsed → try half-open probe
      if (Date.now() - this.lastFailureTime >= this.resetTimeout) {
        this.state = STATES.HALF_OPEN;
        this.halfOpenCalls = 0;
      } else {
        // Fast fail — return fallback immediately
        if (typeof fallback === 'function') return fallback();
        throw new Error('Circuit breaker OPEN — service unavailable');
      }
    }

    if (this.state === STATES.HALF_OPEN && this.halfOpenCalls >= this.halfOpenMaxCalls) {
      // Already have a probe in flight
      if (typeof fallback === 'function') return fallback();
      throw new Error('Circuit breaker HALF_OPEN — waiting for probe result');
    }

    if (this.state === STATES.HALF_OPEN) this.halfOpenCalls++;

    try {
      const result = await fn();
      this._onSuccess();
      return result;
    } catch (err) {
      this._onFailure();
      if (typeof fallback === 'function') return fallback();
      throw err;
    }
  }

  _onSuccess() {
    this.callWindow.push(true);
    if (this.callWindow.length > this.windowSize) this.callWindow.shift();

    if (this.state === STATES.HALF_OPEN) {
      // Probe succeeded — service recovered, reset to closed
      console.log('Circuit breaker: probe succeeded, resetting to CLOSED');
      this.state        = STATES.CLOSED;
      this.failureCount = 0;
      this.callWindow   = [];
    }
  }

  _onFailure() {
    this.callWindow.push(false);
    if (this.callWindow.length > this.windowSize) this.callWindow.shift();
    this.failureCount++;
    this.lastFailureTime = Date.now();

    const failureRate = this.callWindow.filter(r => !r).length / this.callWindow.length * 100;

    if (
      this.state === STATES.HALF_OPEN ||
      (this.callWindow.length >= this.windowSize && failureRate >= this.failureRatePercent) ||
      this.failureCount >= this.failureThreshold
    ) {
      console.warn(`Circuit breaker OPENED (failure rate: ${failureRate.toFixed(0)}%)`);
      this.state = STATES.OPEN;
    }
  }

  getState() {
    return {
      state: this.state,
      failureCount: this.failureCount,
      failureRate: this.callWindow.length
        ? `${(this.callWindow.filter(r=>!r).length / this.callWindow.length * 100).toFixed(0)}%`
        : '0%'
    };
  }
}

// Usage — wrap every external service call
const paymentBreaker = new CircuitBreaker({
  failureThreshold:   5,
  failureRatePercent: 50,
  resetTimeout:       30000,
  windowSize:         20,
});

async function processPayment(order) {
  return paymentBreaker.call(
    // The actual call
    () => paymentAPI.charge(order.userId, order.amount),
    // Fallback when circuit is OPEN
    () => {
      // Queue for later processing instead of failing hard
      paymentQueue.add('retry-payment', order, { delay: 30000 });
      return { status: 'queued', message: 'Payment queued for processing' };
    }
  );
}

// Expose circuit state for monitoring dashboards
app.get('/internal/circuit-breakers', (req, res) => {
  res.json({
    payment: paymentBreaker.getState(),
    // ...other breakers
  });
});
Real World

Netflix Hystrix — the pattern that popularized circuit breakers

Netflix open-sourced Hystrix in 2012. By 2015, every Netflix service had Hystrix wrapping every downstream call. The insight from Netflix: at their scale (millions of users), any downstream call will fail some percentage of the time. Without circuit breakers, rare failures accumulate into thread pool exhaustion, which turns into cascading timeouts, which turns into a full outage.

Netflix discovered that adding fallbacks was equally important. A Hystrix command without a fallback just fails fast. A Hystrix command with a fallback degrades gracefully — the user sees a slightly degraded experience instead of an error page. "Degraded but available" is almost always better than "completely unavailable".

Timeouts are the other half of the pattern. A circuit breaker won't help if your downstream call never returns — it will just sit there, holding a thread. Always pair a circuit breaker with an explicit timeout. The typical pattern: set timeout to 2× the p99 latency of the downstream service, and open the circuit after 5 consecutive timeouts.

Pros

  • Prevents cascading failures across service graph
  • Gives failing services time to recover (no traffic pressure)
  • Fast-fail response instead of slow hang
  • Enables graceful degradation via fallbacks
  • Exposes service health via circuit state metrics

Cons

  • Adds complexity — more moving parts, more config to tune
  • Thresholds require calibration per service and environment
  • Risk of opening on valid load spike, not actual failure
  • Fallback logic must be written and tested
  • Distributed state if circuit state needs to be shared across instances

Key Takeaway

The circuit breaker is the resilience pattern for microservices. Its job is to detect that a downstream service is struggling and stop making the situation worse by continuing to send traffic. The state machine — Closed → Open → Half-Open → Closed — automates failure detection and recovery. Always pair it with a meaningful fallback: fail fast with a queued retry, serve cached data, or return a partial response. A circuit breaker without a fallback just changes the error message; a circuit breaker with a fallback enables graceful degradation.
