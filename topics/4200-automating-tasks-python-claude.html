The three automation types that convert

What businesses actually pay for

1. Document processing and extraction

Businesses receive dozens to hundreds of PDFs, invoices, or email attachments per week that someone reads manually and enters into a spreadsheet. A Python + Claude pipeline — pdfplumber or pypdf for extraction, Claude for structured parsing, Pandas for output — eliminates this entirely.

Pricing anchors: what does the manual process cost in staff time per month? A VA at $20/hour spending 15 hours/week on data entry = $1,200/month. Charge $800 one-time + $200/month maintenance. The client saves money immediately.

2. Research and competitive intelligence reports

Marketing and sales teams spend hours per week manually checking competitor pricing, monitoring news, or compiling market data. An automated pipeline — requests + BeautifulSoup for scraping, Claude for synthesis and formatting — generates a structured weekly report without human involvement.

These convert well as recurring subscriptions ($300–$600/month) because the value compounds over time: the client stops paying a staff member to do the same task.

3. Lead qualification and CRM enrichment

Inbound leads often come as form submissions with incomplete information. A Python script that enriches each lead with public data (LinkedIn via scraping or API, company size from Clearbit, website analysis), passes it through Claude for qualification scoring, and writes back to the CRM saves the sales team from wasting calls on unqualified prospects.

Recurring value makes this a subscription product ($500–$1,500/month depending on volume).

The technical stack

What you actually need to build these

  • Anthropic SDK: pip install anthropic. The Claude API for document parsing, synthesis, and structured extraction. Use structured output (JSON mode) so responses are reliably parseable.
  • Requests + BeautifulSoup / Playwright: Web data gathering. BeautifulSoup for static pages; Playwright for JavaScript-rendered content.
  • Pandas: Cleaning, transforming, and exporting data to CSV or Excel.
  • pdfplumber: Extracting text and tables from PDFs with layout awareness.
  • APScheduler: Running scripts on a schedule (daily, weekly). Turns a one-off script into a recurring service.
  • smtplib / sendgrid: Emailing the output report directly to the client.
  • Fly.io or Railway: Hosting the scheduler cheaply ($5–$10/month). The client does not need to run anything locally.

Minimal viable invoice processor (structure)

import anthropic, pdfplumber, json, pandas as pd

client = anthropic.Anthropic()

def extract_invoice(pdf_path):
    with pdfplumber.open(pdf_path) as pdf:
        text = "\n".join(page.extract_text() for page in pdf.pages)

    response = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=512,
        messages=[{
            "role": "user",
            "content": f"Extract invoice data as JSON with keys: vendor, date, total, line_items.\n\n{text}"
        }]
    )
    return json.loads(response.content[0].text)

Client acquisition

How to find the first three clients

  • Start with your network. Think of every small business or department you know. Ask one question: "What is something your team does manually every week that you wish could just happen automatically?" You will get answers immediately.
  • Offer a free pilot. Build a prototype for one real use case at no cost. Seeing the output of their own data converted into a formatted report is more persuasive than any pitch deck.
  • Charge for the outcome, not the code. "We saved you 12 hours/week" is worth $800/month to a small business. "I wrote a 200-line Python script" is worth $0. Frame every proposal in terms of time saved or revenue generated.
  • Upwork and Contra for first clients: Search for "data entry automation," "PDF extraction," "web scraping reports." These job postings are looking for exactly this skill set. Win two projects; use them as case studies.
Common questions

FAQ

  • Do clients care that you're using Claude behind the scenes?

    Generally no — they care about the output. Be transparent if asked, and include the API cost in your pricing (it is usually small: $10–30/month for most automations at typical business volumes).

  • What Python skill level do you need?

    You need to be comfortable writing scripts that read files, call APIs, and export data. You do not need to understand decorators, metaclasses, or advanced OOP. The Python skill level required is genuinely beginner-to-intermediate.

  • Is this sustainable long-term or will it commoditise?

    The commodity risk is real for simple one-off scripts. The sustainable version is becoming the person who understands a specific client's workflow deeply — which no tool automates away. Specialising in one industry (real estate, legal, logistics) builds moat that generic automation tools don't have.

Key Takeaway

The technical bar for automation that businesses will pay for is lower than most developers think. The skill gap is not Python syntax — it is identifying which manual processes have enough repetition and pain to justify automation, and pricing based on value delivered rather than code written.
