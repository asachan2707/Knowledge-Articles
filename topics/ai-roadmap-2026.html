AI engineering in 2026 is not about knowing every paper — it is about knowing which tools to reach for, how to evaluate model outputs, how to design systems that degrade gracefully when models are wrong, and how to ship and maintain AI features in production codebases. Core idea: Learn AI in the order problems appear: foundations → prompting → retrieval → agents → infrastructure → specialisation. Each phase builds on the last, and each has a concrete "graduation project" that proves the skill is real.

Phase 1

Foundations — what you must know before anything else

JavaScript for AI (recommended path)

JavaScript/Node.js is the fastest path to shipping AI features because the same language runs the AI call and the web API. You need:

  • async/await and fetch — every AI API is an HTTP call. You must handle promises fluently.
  • JSON manipulation — AI responses are JSON. Map, filter, destructure with confidence.
  • npm packages — the AI SDK ecosystem is npm-first.
  • Streams — streaming AI responses requires Node.js streams or ReadableStream.

What an LLM actually is (the minimum mental model)

  • An LLM predicts the next token. It does not "know" things — it completes text based on training patterns.
  • Context window = everything the model can see at once. Think of it as short-term memory.
  • Temperature controls randomness. 0 = deterministic, 1 = creative, >1 = chaotic.
  • Tokens ≠ words. English averages ~0.75 tokens/word. Cost = tokens × price-per-token.

Graduation project: Build a CLI that reads a file and summarises it using the Anthropic API.

Phase 2

LLMs & Prompting — calling models the right way

The Anthropic SDK in 90 lines

import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic(); // reads ANTHROPIC_API_KEY from env

// 1. Basic call
const basic = await client.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 1024,
  messages: [{ role: 'user', content: 'Explain vector embeddings in one paragraph.' }],
});
console.log(basic.content[0].text);

// 2. System prompt
const withSystem = await client.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 512,
  system: 'You are a senior software architect. Reply concisely with trade-offs.',
  messages: [{ role: 'user', content: 'Should I use Redis or Postgres for a job queue?' }],
});

// 3. Streaming (for UI responsiveness)
const stream = client.messages.stream({
  model: 'claude-sonnet-4-6',
  max_tokens: 2048,
  messages: [{ role: 'user', content: 'Write a Node.js rate limiter middleware.' }],
});
for await (const chunk of stream) {
  if (chunk.type === 'content_block_delta') {
    process.stdout.write(chunk.delta.text ?? '');
  }
}

// 4. Structured output via tool use
const structured = await client.messages.create({
  model: 'claude-sonnet-4-6',
  max_tokens: 512,
  tools: [{
    name: 'extract_issue',
    description: 'Extract structured bug report data',
    input_schema: {
      type: 'object',
      properties: {
        title:    { type: 'string' },
        severity: { type: 'string', enum: ['low','medium','high','critical'] },
        steps:    { type: 'array', items: { type: 'string' } },
      },
      required: ['title', 'severity', 'steps'],
    },
  }],
  tool_choice: { type: 'tool', name: 'extract_issue' },
  messages: [{ role: 'user', content: 'The app crashes when you click Save after uploading an image larger than 5MB.' }],
});
const issue = structured.content[0].input; // typed object

Key prompting patterns

  • Role + task + format: "You are X. Do Y. Return Z." — the three-part prompt template.
  • Few-shot examples: 2–3 worked examples in the prompt improve consistency dramatically.
  • Chain-of-thought: "Think step by step" unlocks better reasoning on multi-step problems.
  • Negative constraints: "Do not include X" is more reliable than trying to describe the absence.
  • Output schema anchoring: describe the exact JSON shape you want before the model fills it.

Graduation project: Build a resume analyser that extracts structured data (skills, experience years, education) using tool use.

Phase 3

RAG & Knowledge — giving models access to your data

Embeddings — the foundation of retrieval

An embedding is a vector (array of floats) that encodes semantic meaning. Similar concepts cluster near each other in this high-dimensional space.

import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

// Get embeddings for a list of texts
async function embed(texts) {
  const res = await client.embeddings.create({
    model: 'voyage-3-large',   // Anthropic's voyage embedding models
    input: texts,
    input_type: 'document',
  });
  return res.data.map(d => d.embedding); // float32[][]
}

// Cosine similarity
function cosineSim(a, b) {
  const dot  = a.reduce((s, v, i) => s + v * b[i], 0);
  const normA = Math.sqrt(a.reduce((s, v) => s + v * v, 0));
  const normB = Math.sqrt(b.reduce((s, v) => s + v * v, 0));
  return dot / (normA * normB);
}

// Naive in-memory vector store
class VectorStore {
  #docs = [];

  async add(texts) {
    const vecs = await embed(texts);
    texts.forEach((text, i) => this.#docs.push({ text, vec: vecs[i] }));
  }

  async search(query, topK = 3) {
    const [qVec] = await embed([query]);
    return this.#docs
      .map(d => ({ ...d, score: cosineSim(qVec, d.vec) }))
      .sort((a, b) => b.score - a.score)
      .slice(0, topK);
  }
}

// Use it
const store = new VectorStore();
await store.add([
  'Redis is an in-memory key-value store often used for caching.',
  'PostgreSQL is a relational database with ACID guarantees.',
  'Kafka is a distributed event streaming platform.',
]);
const hits = await store.search('what should I use for a job queue?');
console.log(hits.map(h => h.text));

Core RAG sub-topics

  • Chunking: fixed-size (512 tokens), recursive (by paragraph/sentence), semantic (by topic). Chunk size is the biggest quality lever.
  • Hybrid search: combine dense (embeddings) + sparse (BM25 keyword) for better recall. Critical for technical queries.
  • Re-ranking: use a cross-encoder to re-score top-50 candidates before passing top-5 to the LLM.
  • Context window management: retrieved chunks compete with conversation history. Budget explicitly.
  • Citation and grounding: ask the model to cite its sources — dramatically reduces hallucination.
  • Production stores: pgvector (Postgres extension), Pinecone, Weaviate, Qdrant, Chroma.

Graduation project: Build a Q&A bot over a 100-page PDF using embeddings + hybrid search + citations.

Phase 4

Agents & Tools — models that act, not just answer

The ReAct loop in code

// Minimal ReAct agent with tool use
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

const TOOLS = [
  {
    name: 'search_web',
    description: 'Search the web for current information',
    input_schema: { type: 'object', properties: { query: { type: 'string' } }, required: ['query'] },
  },
  {
    name: 'run_code',
    description: 'Execute JavaScript code and return the result',
    input_schema: { type: 'object', properties: { code: { type: 'string' } }, required: ['code'] },
  },
];

// Tool implementations
function searchWeb({ query })  { return `[search results for: ${query}]`; } // stub
function runCode({ code })     { try { return String(eval(code)); } catch(e) { return `Error: ${e.message}`; } }

async function runAgent(userMessage) {
  const messages = [{ role: 'user', content: userMessage }];

  while (true) {
    const response = await client.messages.create({
      model: 'claude-sonnet-4-6',
      max_tokens: 4096,
      tools: TOOLS,
      messages,
    });

    messages.push({ role: 'assistant', content: response.content });

    if (response.stop_reason === 'end_turn') {
      return response.content.find(b => b.type === 'text')?.text;
    }

    if (response.stop_reason === 'tool_use') {
      const toolResults = response.content
        .filter(b => b.type === 'tool_use')
        .map(block => {
          const result = block.name === 'search_web' ? searchWeb(block.input)
                       : block.name === 'run_code'   ? runCode(block.input)
                       : 'Unknown tool';
          console.log(`Tool: ${block.name} → ${result}`);
          return { type: 'tool_result', tool_use_id: block.id, content: result };
        });

      messages.push({ role: 'user', content: toolResults });
    }
  }
}

const answer = await runAgent('What is 2^32 + the number of days in a leap year?');
console.log(answer);

Agent design principles

  • Tools over instructions: give the agent a tool instead of describing what to do manually.
  • Typed tool outputs: tools that return structured JSON are more reliable than free text.
  • Risk gates: classify tool calls by risk (read / write / execute / external). Gate higher risk with confirmation.
  • Max iterations: always cap the agent loop. Unbounded agents loop on failures.
  • MCP servers: the standard protocol for plugging third-party tools (GitHub, Slack, Postgres) into any agent.

Graduation project: Build a coding agent that reads a GitHub issue, writes a fix, and opens a PR.

Phase 5

Infrastructure — making AI systems production-grade

Evals — the non-negotiable

Evals are automated tests for AI behaviour. Without them, every model upgrade is a gamble.

// Simple eval harness
const TEST_CASES = [
  { input: 'What is 2+2?',                   expected: '4' },
  { input: 'Capital of France?',             expected: 'Paris' },
  { input: 'Is "hello" a palindrome?',       expected: 'no' },
];

async function runEvals(model) {
  let pass = 0;
  for (const { input, expected } of TEST_CASES) {
    const res = await client.messages.create({
      model,
      max_tokens: 64,
      messages: [{ role: 'user', content: input }],
    });
    const output = res.content[0].text.toLowerCase();
    const ok = output.includes(expected.toLowerCase());
    if (ok) pass++;
    console.log(`${ok ? '✓' : '✗'} "${input}" → got "${output.slice(0,60)}"`);
  }
  return { pass, total: TEST_CASES.length, score: pass / TEST_CASES.length };
}

const score = await runEvals('claude-haiku-4-5-20251001');
console.log(`Score: ${(score.score * 100).toFixed(0)}%`);

Key infrastructure sub-topics

  • Tracing: LangSmith, Langfuse, or custom spans — log every prompt/response with latency and cost.
  • Guardrails: input validation, output filtering, topic blocking. Implement as middleware around every LLM call.
  • Caching: semantic caching (cache by embedding similarity) cuts repeat costs by 60–80% for FAQ-style queries.
  • Rate limiting: per-user token budgets prevent abuse and cost explosions.
  • Fallback chains: expensive model → cheaper model → static response. Design for graceful degradation.
  • Deployment: containerise with Docker, serve behind a gateway (Kong, AWS API Gateway), use feature flags for model rollouts.

Graduation project: Add tracing, evals CI job, and semantic caching to your Phase 3 RAG project.

Tools and resources

The 2026 AI engineer toolkit

  • SDKs: @anthropic-ai/sdk, openai — always prefer official SDKs over raw fetch.
  • Orchestration: LangChain.js, LlamaIndex.TS, or build your own (simple is often better).
  • Vector stores: pgvector (if already on Postgres), Qdrant (self-hosted), Pinecone (managed).
  • Tracing: Langfuse (open source), LangSmith, or Helicone for cost tracking.
  • Local models: Ollama for running Llama 3, Mistral, Phi-4 locally for dev/test.
  • Evals: PromptFoo, Braintrust, or write your own harness.
  • Deployment: Vercel AI SDK for Next.js, Modal for serverless GPU, Fly.io for containers.
