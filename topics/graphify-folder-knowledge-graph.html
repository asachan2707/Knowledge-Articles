How to Use Graphify: Turn Any Folder Into a Knowledge Graph

Graphify is an open-source skill that converts a folder of code, docs, PDFs, images, and videos into a queryable knowledge graph — and reportedly cuts token usage by up to 71.5x compared to dumping everything into context.

Portal Medium · Agentic Builders
Author Ana Bildea, PhD
Published Apr, 2026
Focus Code understanding, knowledge extraction, AI coding assistants

Graphify takes a folder — any folder — and turns it into a structured knowledge graph that AI assistants can query. Instead of pasting whole files into Claude or Cursor and hoping the model reads what matters, you give it a graph that already knows the call relationships, central modules, and surprising cross-file connections.

Core idea: the bottleneck for AI coding assistants is rarely the model — it is the way the codebase is presented to it. A graph is a much better presentation than a pile of files.

Why this tool got attention

Most "AI for codebases" tools either embed everything and hope vector search picks the right chunks, or dump huge swaths of files into the prompt. Graphify takes a third path: build the graph once, then let the assistant navigate it. The reported 71.5x token reduction is the headline, but the more interesting claim is that the assistant gives better answers when it can see structure instead of text.

Installation

One command to install

Graphify needs Python 3.10+. The PyPI package name is graphifyy (double-y), but the CLI is graphify: 

pip install graphifyy && graphify install

Installation supports a long list of host environments — Claude Code, Codex, OpenCode, GitHub Copilot CLI, VS Code Copilot Chat, Cursor, Gemini CLI, and others — so you pick whichever AI assistant you already use.

Basic usage

Build a graph in one command

From inside any folder you want indexed: 

/graphify .

That produces a graphify-out/ directory with three artifacts:

  • graph.html — interactive visualization. Click nodes, filter, search.
  • GRAPH_REPORT.md — concepts, surprising connections, suggested questions to ask the graph.
  • graph.json — persistent, queryable graph for the AI assistant to use later.
How it works under the hood

Multi-stage pipeline, not a single embedding pass

  • Tree-sitter extracts code syntax trees and call graphs.
  • LLM-driven semantic extraction pulls concepts from prose, comments, and diagrams.
  • NetworkX + Leiden clusters the result into communities of related nodes.
  • "God node" detection highlights high-degree central files so you know where leverage lives.
  • Cross-file connection surfacing exposes links between modules that text search would miss.
Where it pays off

Use cases that match the design

  • Onboarding to an unfamiliar large repo — read GRAPH_REPORT.md first, not the files.
  • Refactor planning — see which modules are central and which are leaves.
  • Documentation generation — anchor docs to actual structure, not vibes.
  • Cross-format research folders — combine code, PDFs, and notes in one graph.
  • Long-running agent sessions — query the graph instead of re-reading files.
Common questions

FAQ for first-time users

  • Is the 71.5x token reduction realistic?

    It's a best-case figure on a sufficiently large codebase where most files were previously being loaded just-in-case. On small projects the savings are modest; on large ones the structure-first approach compounds.

  • Does it support non-code folders?

    Yes — PDFs, images, video, and prose are all extracted via the LLM-driven stage and added to the same graph.

  • Do I have to re-run it after every change?

    For active development you re-build periodically. The graph is meant as a navigation surface, not a live index — pair it with normal file reads for hot paths.

Key Takeaway

Graphify is a structural answer to "how do I let an AI assistant understand my codebase without paying for it every time?" Build the graph once, let the assistant navigate it, and the same model gives sharper answers while spending dramatically fewer tokens. That tradeoff is exactly the direction agent tooling is heading.
