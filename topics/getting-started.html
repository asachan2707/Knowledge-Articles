Getting Started

This page is the writing pattern used in the Learning project. Topic pages should read like newsletter articles or study notes — long enough to teach you something on a re-read, short enough to scan.

A good topic page should help you understand the meaning behind an article, not just remember its title. The goal is to turn scattered reading into clear personal knowledge that is pleasant to revisit later.

Core idea: capture the essence of what you read, in your own words, with enough detail that future-you can rebuild the idea without going back to the original source.

Story · Why a hub like this exists

Most reading evaporates within a week. You bookmark an article, skim it, feel smart for ten minutes, and never think about it again. A few months later, you cannot remember which article had the idea you wanted to use, let alone what the idea was.

A personal Learning Hub fixes this with one move: the act of writing the summary forces understanding, and the act of organizing it into one searchable place makes it usable later. The output is not a blog. It is a private library you actually trust.

Page anatomy

The newsletter-style template every page uses

  1. Hero image + kicker: a relevant image and a short label (category · read time) for quick visual identification.
  2. Title and summary: the article's name in plain language plus a 2-3 sentence summary.
  3. Article metadata: source portal, author, publication, date — for crediting and recall.
  4. Lead + core idea callout: the central insight in one paragraph, then a one-line core idea pulled out as a callout.
  5. Story card: a real-world anecdote, scenario, or example that anchors the abstract idea.
  6. Detailed breakdown: sections with mini-headings explaining the concept, often numbered.
  7. Practical learning: a checklist of how to apply this in real work.
  8. FAQ / common questions: the pushbacks and what-ifs the article anticipates.
  9. Key takeaway: one paragraph the reader can keep in their head a year later.
Writing guidance

How to make pages reader-friendly

  • Prefer short paragraphs that carry one main idea each.
  • Explain in your own words instead of repeating headlines.
  • Use section labels and mini-headings to break up longer explanations.
  • Include practical implications — what to actually do with the idea.
  • Add a story or example so the abstract becomes memorable.
  • Write enough detail that revisiting the page later still teaches you something.
Categories and tags

Use the home page as the entry point

The hub's home page (index.html) is a filterable index. Each topic card carries category badges (AI, Python, System Design, Career, Productivity, Finance, Tools, Learning, Project, Meta) and a searchable keyword list. Filter by category, search by keyword, and find the topic you need without scrolling through everything.

When you add a new topic, set its categories on the topic card in index.html and include relevant keywords. The filter system will pick it up automatically.

Workflow

How to add a new topic

  1. Copy any existing topic file in topics/ as a template.
  2. Update the title, summary, metadata, hero image, and content sections.
  3. Pick an Unsplash image URL or use a CSS gradient hero — both are supported.
  4. Add a new card in index.html under the topic grid with the right category badges and keywords.
  5. Open the home page; the new topic should appear with the correct category counts updated automatically.

Key Takeaway

The hub works because every topic uses the same shape: hero, story, breakdown, practice, FAQ, takeaway. Once you internalize the template, writing a new page is fast and reading any page is easy. That consistency is the whole reason the hub stays useful as it grows.
