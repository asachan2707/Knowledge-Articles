Claude Desktop Has a Complete AI Workspace Hidden Inside It

Most users treat Claude Desktop as a chat window. It is actually a full AI workspace — with Projects, persistent memory, integrated MCP connections, computer use, and a background agent layer that most users never discover.

Portal Internal · Curated
Focus Claude Desktop hidden features: Projects, memory, MCP, computer use, background agents
Published Jun 2026

Claude Desktop ships with a user interface that looks like a chat app. Underneath it is a complete AI workspace: project-scoped memory, MCP server connections that give Claude access to your local files and tools, a computer use layer that lets Claude control your desktop, and a background agent system that runs tasks without keeping a window open. Most users interact with 20% of what is available.

Core idea: The gap between "Claude as chatbot" and "Claude as AI workspace" is almost entirely a feature discovery problem. The workspace features are already there — they just require one-time configuration to unlock.

Story · The setup that changed everything

The typical discovery path: a developer uses Claude Desktop for months as a chat interface, then stumbles onto the MCP configuration panel, connects their filesystem and a database server, and realises Claude can now read their actual project files without copy-paste. The productivity change is immediate and significant. None of this required installing anything new — the MCP panel was there the whole time.

Feature 1: Projects

Persistent, scoped workspaces with memory

Projects give every conversation a persistent context: a project-level system prompt, uploaded knowledge files, and memory that persists across sessions. The chat history is scoped to the project. Claude remembers your preferences, your codebase structure, your naming conventions, and decisions made in prior sessions — without you re-explaining them every time.

  • Project instructions: Equivalent to a CLAUDE.md but for the Desktop app — Claude reads them at the start of every project conversation.
  • Knowledge files: Upload PDFs, docs, or text files that Claude references throughout the project. API documentation, style guides, architecture docs — persistent, not re-uploaded each session.
  • Conversation history: All prior conversations within a project are searchable and referenced by Claude when relevant.
Feature 2: MCP Connections

Giving Claude access to your actual environment

The Model Context Protocol (MCP) panel in Desktop settings lets you connect Claude to external tools and data sources. Once connected, Claude can read files, query databases, search the web, run code, and call APIs — from inside a Desktop conversation.

The most useful MCP connections to add immediately

  • Filesystem: Read and write files on your local machine. Claude can read your codebase without you pasting code into the chat.
  • Browser: Claude can search the web, read URLs, and retrieve documentation on demand rather than working from its training data.
  • Code execution: Run Python, JavaScript, or shell commands. Claude can test its own suggestions.
  • Notion / Google Drive: Access your notes, documents, and project files directly from conversation.
  • GitHub: Read repositories, issues, PRs, and code without switching apps.

Configuration is a JSON file (~/.claude/mcp_settings.json or the Desktop settings panel). Each server takes 5–10 minutes to add and changes what Claude can do fundamentally.

Feature 3: Computer Use

Claude controlling your desktop

Computer use lets Claude take screenshots of your screen, move the mouse, click, type, and execute keyboard shortcuts — treating your desktop as a tool. Practical use cases:

  • Filling out repetitive forms in applications that have no API.
  • Extracting data from desktop software by reading the screen.
  • Automating sequences in GUI-based tools (design software, legacy enterprise apps).
  • Taking periodic screenshots to monitor a process and alert when something changes.

Computer use requires explicit activation and runs with your desktop permissions. It is more powerful and more risk-bearing than MCP connections — review what you are authorising before running unattended computer use workflows.

Feature 4: Background agents

Long-running tasks without keeping a window open

Background agents let you kick off a task — "research these 20 companies and compile a report," "refactor these five files to use the new API pattern," "generate documentation for every public function in this module" — and close the window. The task runs in the background and notifies you when complete.

The key difference from a standard conversation: background agents can run for minutes to hours, take multiple tool-calling steps, and produce a structured artifact as their output. You receive a notification with a summary and the result artifact when the task finishes.

Feature 5: Artifacts and canvas

Outputs that live beyond the chat

When Claude generates a document, code file, spreadsheet, or diagram, it can render it as a live Artifact — a panel alongside the conversation that shows the output in a rendered form and can be edited, exported, or iterated on. Artifacts persist in the project and can be opened, modified, and reshared independently of the conversation.

The canvas mode (available for certain artifact types) lets you work collaboratively with Claude on a document or code file directly — edits happen in the rendered view, not in a code block.

Getting started

The one-hour setup that unlocks the workspace

  1. Create one Project for your most active work context. Add a project instruction paragraph explaining who you are and what conventions to follow.
  2. Connect the filesystem MCP to your main project directory. Test it by asking Claude to list the files in a specific folder.
  3. Add a browser or web search MCP so Claude can retrieve current documentation without you pasting URLs.
  4. Upload one knowledge file — an architecture doc, an API reference, or a style guide — to the Project knowledge base. Ask Claude a question that requires it.
  5. Try one background agent task — something that would take you 30 minutes manually. Observe the result.
Common questions

FAQ

  • Is Claude Desktop the same as Claude Code?

    No. Claude Code is a CLI tool for software engineers — it runs in your terminal and integrates with your codebase. Claude Desktop is a full-featured GUI application with Projects, MCP connections, computer use, and artifacts. They share the Claude model but target different workflows. Both can connect MCP servers.

  • Does enabling MCP connections create security risks?

    MCP connections run with your local credentials and file permissions. A filesystem MCP gives Claude the same read/write access you have. Review each server before connecting it and restrict the allowed paths in the configuration. The risk is proportional to what you connect.

  • Is the Projects memory truly persistent between sessions?

    Yes. Project instructions, knowledge files, and conversation history persist indefinitely within the project. Claude references prior conversations when relevant to the current one. This is distinct from the per-conversation window — it is a genuine long-term memory layer.

Key Takeaway

Claude Desktop is not a chat app. It is an AI workspace with persistent project memory, MCP-connected access to your tools and files, computer use, and a background agent layer. The one-hour setup to unlock these features is worth more than a year of prompting more cleverly inside a bare chat window.
