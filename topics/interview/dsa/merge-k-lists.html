Merge K Sorted Lists

Merge k sorted linked lists into one using a min-heap for O(n log k) extraction, or divide-and-conquer for the same complexity without a heap.

DifficultyHard
PatternMin Heap / Divide & Conquer
CompaniesGoogle · Amazon · Microsoft · Meta
LanguageJavaScript (ES2020+)
Problem Statement

Description

You are given an array of k linked lists, each sorted in ascending order. Merge all of them into one sorted linked list and return it.

Example: lists = [[1→4→5],[1→3→4],[2→6]] → 1→1→2→3→4→4→5→6.

Approach

Core Insight & Strategy

Core insight: At any point, the next node in the merged list is the globally minimum head across all k lists. A min-heap of size k lets you extract that minimum in O(log k) and push its successor. Total cost is O(n log k) where n is the total number of nodes. JavaScript has no built-in heap, so implement a simple min-heap or use the divide-and-conquer approach (no heap needed).

Step-by-step (divide and conquer)

  1. Repeatedly pair up adjacent lists and merge each pair (2-list merge in O(a+b)).
  2. After each round the list count halves. After log k rounds all lists are merged.
  3. Total work per round = O(n); total = O(n log k).
Flow Diagram

Operation trace — lists=[[1→4→5],[1→3→4],[2→6]]

k=3 lists: L0: 1→4→5 L1: 1→3→4 L2: 2→6 Divide & Conquer Round 1 (k=3 → pair up): merge(L0, L1): compare heads, pick smaller each step 1vs1 → 1(L0) 1vs1 → 1(L1) 3vs4 → 3 4vs4 → 4(L0) 4vs5 → 4(L1) 5 L01: 1→1→3→4→4→5 L2 unpaired → carries over as-is Round 2 (merge L01 with L2): 1vs2 → 1 1vs2 → 1 2vs3 → 2 3vs6 → 3 4vs6 → 4 4vs6 → 4 5vs6 → 5 6 Result: 1→1→2→3→4→4→5→6 Min-heap approach: Init heap with first node of each list. Each extraction: pop min (O(log k)), append to result, push popped node's .next. Total n extractions × O(log k) = O(n log k)
JavaScript Solution

Code

class ListNode {
  constructor(val, next = null) { this.val = val; this.next = next; }
}

// Divide & Conquer — no heap needed
function mergeKLists(lists) {
  if (!lists.length) return null;
  while (lists.length > 1) {
    const merged = [];
    for (let i = 0; i < lists.length; i += 2) {
      merged.push(mergeTwoLists(lists[i], lists[i + 1] ?? null));
    }
    lists = merged;
  }
  return lists[0];
}

function mergeTwoLists(l1, l2) {
  const dummy = new ListNode(0);
  let curr = dummy;
  while (l1 && l2) {
    if (l1.val <= l2.val) { curr.next = l1; l1 = l1.next; }
    else                   { curr.next = l2; l2 = l2.next; }
    curr = curr.next;
  }
  curr.next = l1 ?? l2;
  return dummy.next;
}

// Min-heap approach (using a simple array-based min-heap)
class MinHeap {
  constructor() { this.h = []; }
  push(node) {
    this.h.push(node);
    let i = this.h.length - 1;
    while (i > 0) {
      const p = (i - 1) >> 1;
      if (this.h[p].val <= this.h[i].val) break;
      [this.h[p], this.h[i]] = [this.h[i], this.h[p]];
      i = p;
    }
  }
  pop() {
    const top = this.h[0];
    const last = this.h.pop();
    if (this.h.length) {
      this.h[0] = last;
      let i = 0;
      while (true) {
        let smallest = i;
        const l = 2*i+1, r = 2*i+2;
        if (l < this.h.length && this.h[l].val < this.h[smallest].val) smallest = l;
        if (r < this.h.length && this.h[r].val < this.h[smallest].val) smallest = r;
        if (smallest === i) break;
        [this.h[i], this.h[smallest]] = [this.h[smallest], this.h[i]];
        i = smallest;
      }
    }
    return top;
  }
  get size() { return this.h.length; }
}

function mergeKListsHeap(lists) {
  const heap = new MinHeap();
  for (const head of lists) if (head) heap.push(head);
  const dummy = new ListNode(0);
  let curr = dummy;
  while (heap.size) {
    const node = heap.pop();
    curr.next = node;
    curr = curr.next;
    if (node.next) heap.push(node.next);
  }
  return dummy.next;
}
Time: O(n log k) both approaches Space: O(log k) recursion for D&C, O(k) for heap
