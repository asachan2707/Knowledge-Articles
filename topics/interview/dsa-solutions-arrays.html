Problem 1 · Easy · Hash Map

Two Sum

Given an integer array nums and an integer target, return the indices of the two numbers that add up to target. Exactly one solution exists; the same element cannot be used twice.

Core insight: instead of scanning every pair (O(n²)), store each number's index in a hash map as you go. For each new number, check if its complement (target - num) already exists in the map. One pass, O(n).

Flow diagram — tracing nums=[2,7,11,15], target=9

nums = [2, 7, 11, 15] target = 9 seen = {} (hash map: value → index) Step 1: i=0, num=2 complement = 9 - 2 = 7 Is 7 in seen? No Store: seen = {2: 0} Step 2: i=1, num=7 complement = 9 - 7 = 2 Is 2 in seen? YES ← seen[2] = 0 Return [seen[2], i] = [0, 1] ────────────────────────────────── If no complement found, continue to next element. Map grows: each element added only once → O(n) time, O(n) space. 
def two_sum(nums: list[int], target: int) -> list[int]:
    seen = {}                          # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []                          # guaranteed to find a solution
Time: O(n) Space: O(n)

Similar problems this pattern solves

  • Three Sum — fix one pointer, run Two Sum on the rest
  • Four Sum — fix two pointers, run Two Sum on the rest
  • Two Sum II (sorted) — use two-pointer instead of hash map (O(1) space)
Problem 2 · Easy · Greedy / Single Pass

Best Time to Buy & Sell Stock

Given an array prices where prices[i] is the price of a stock on day i, find the maximum profit from a single buy-then-sell transaction. Return 0 if no profit is possible.

Core insight: scan left to right. Track the minimum price seen so far (the best buy day). At each new day, compute price - min_so_far. The maximum of all these values is the answer — you never need to look back further than the current minimum.

Flow diagram — prices=[7,1,5,3,6,4]

prices = [7, 1, 5, 3, 6, 4] min_price = ∞ max_profit = 0 Day 0: price=7 min=7 profit=0-0=0 max_profit=0 Day 1: price=1 min=1 profit=1-1=0 max_profit=0 ← new min Day 2: price=5 min=1 profit=5-1=4 max_profit=4 Day 3: price=3 min=1 profit=3-1=2 max_profit=4 Day 4: price=6 min=1 profit=6-1=5 max_profit=5 ← new best Day 5: price=4 min=1 profit=4-1=3 max_profit=5 Answer: 5 (buy at day 1 price=1, sell at day 4 price=6) 
def max_profit(prices: list[int]) -> int:
    min_price = float('inf')
    max_profit = 0
    for price in prices:
        min_price = min(min_price, price)
        max_profit = max(max_profit, price - min_price)
    return max_profit
Time: O(n) Space: O(1)

Similar problems this pattern solves

  • Stock II (unlimited transactions) — accumulate every positive price difference
  • Stock with cooldown — DP states: held / sold / cooldown
Problem 3 · Medium · Two Pointers

Container With Most Water

Given array height of n integers, find two lines that together with the x-axis form a container that holds the most water. Return the maximum area.

Core insight: place left and right pointers at both ends. Area = min(h[l], h[r]) × (r - l). Moving the pointer at the taller side can never increase the area (the width shrinks, height is already limited by the shorter side). So always move the shorter-side pointer inward.

Flow diagram — height=[1,8,6,2,5,4,8,3,7]

height = [1, 8, 6, 2, 5, 4, 8, 3, 7] 0 1 2 3 4 5 6 7 8 L=0 R=8: min(1,7)×8 = 1×8 = 8 move L (height[L]=1 is shorter) L=1 R=8: min(8,7)×7 = 7×7 = 49 move R (height[R]=7 is shorter) L=1 R=7: min(8,3)×6 = 3×6 = 18 move R L=1 R=6: min(8,8)×5 = 8×5 = 40 move either (tie → move R) L=1 R=5: min(8,4)×4 = 4×4 = 16 move R ... (all subsequent areas smaller) Answer: 49 Why moving the taller side is wrong: At L=1(h=8), R=8(h=7) → area=49. If we moved L instead: L=2(h=6), R=8(h=7) → area = min(6,7)×6 = 36 < 49 ✗ Moving the taller side can only decrease or maintain; never improves. 
def max_area(height: list[int]) -> int:
    left, right = 0, len(height) - 1
    best = 0
    while left < right:
        area = min(height[left], height[right]) * (right - left)
        best = max(best, area)
        if height[left] <= height[right]:
            left += 1
        else:
            right -= 1
    return best
Time: O(n) Space: O(1)
Problem 4 · Hard · Two Pointers

Trapping Rain Water

Given an elevation map represented by array height, compute how much water it can trap after raining.

Core insight: water at any bar = min(max_left, max_right) − height[i]. Use two pointers to avoid the O(n) precomputation of max arrays. Whichever side has the smaller maximum determines the water level at that bar.

Flow diagram — height=[0,1,0,2,1,0,1,3,2,1,2,1]

Elevation map (visualised): # # ## # # ## ### ## [0,1,0,2,1,0,1,3,2,1,2,1] Two-pointer trace: L → starts at 0 R → starts at 11 left_max tracks max seen from left right_max tracks max seen from right Rule: if left_max <= right_max → process left pointer, else process right pointer water at current bar = max_side - height[current] move the pointer inward State: L R left_max right_max water 0 11 0 1 0 L=0: h=0, left_max=0, water += 0-0=0, L→1 L=1: h=1, left_max=1, water += 1-1=0, L→2 L=2: h=0, left_max=1, water += 1-0=1, L→3 R=11: h=1, right_max=1, water += 1-1=0, R→10 R=10: h=2, right_max=2, water += 0, R→9 L=3: h=2, left_max=2, water += 0, L→4 ... (continues) Total trapped: 6 
def trap(height: list[int]) -> int:
    left, right = 0, len(height) - 1
    left_max = right_max = 0
    water = 0
    while left < right:
        if left_max <= right_max:
            left_max = max(left_max, height[left])
            water += left_max - height[left]
            left += 1
        else:
            right_max = max(right_max, height[right])
            water += right_max - height[right]
            right -= 1
    return water
Time: O(n) Space: O(1)

Why this is a Hard problem

The intuition behind the two-pointer approach is non-obvious. The key proof: when left_max ≤ right_max, the water at bar left is determined by left_max alone — we know the right side has at least right_max ≥ left_max so it can hold water up to left_max. We do not need to know the exact right maximum.

Problem 5 · Medium · Sliding Window

Longest Substring Without Repeating Characters

Given string s, find the length of the longest substring that contains no duplicate characters.

Core insight: maintain a sliding window [left, right]. Expand right by one each step. If the new character is already in the window, shrink the window from the left until it's gone. A hash map tracks the most recent index of each character — this lets you jump left directly instead of shrinking one step at a time.

Flow diagram — s="abcabcbb"

s = "a b c a b c b b" 0 1 2 3 4 5 6 7 Window [L..R], char_index={char: last_seen_idx}, max_len=0 R=0: 'a' → new. window=[0..0], char_index={a:0}, max=1 R=1: 'b' → new. window=[0..1], char_index={a:0,b:1}, max=2 R=2: 'c' → new. window=[0..2], char_index={..c:2}, max=3 R=3: 'a' → seen at 0. L = max(L, 0+1) = 1 window=[1..3], char_index={a:3,...} max=3 R=4: 'b' → seen at 1. L = max(L, 1+1) = 2 window=[2..4], char_index={b:4,...} max=3 R=5: 'c' → seen at 2. L = max(L, 2+1) = 3 window=[3..5] max=3 R=6: 'b' → seen at 4. L = max(L, 4+1) = 5 window=[5..6] max=3 R=7: 'b' → seen at 6. L = max(L, 6+1) = 7 window=[7..7] max=3 Answer: 3 (window "abc") 
def length_of_longest_substring(s: str) -> int:
    char_index = {}    # char -> most recent index
    left = 0
    max_len = 0
    for right, ch in enumerate(s):
        if ch in char_index and char_index[ch] >= left:
            left = char_index[ch] + 1       # jump left past duplicate
        char_index[ch] = right
        max_len = max(max_len, right - left + 1)
    return max_len
Time: O(n) Space: O(min(n, charset)) — at most 128 for ASCII

Similar problems this pattern solves

  • Minimum Window Substring — shrink window only when the constraint is met
  • Permutation in String — fixed-size window, character frequency counts
  • Fruit Into Baskets — sliding window with at most k distinct values
Problem 6 · Medium · Prefix Sum + Hash Map

Subarray Sum Equals K

Given an integer array nums and integer k, return the total number of subarrays whose elements sum to k.

Core insight: define prefix[i] = sum of nums[0..i]. A subarray nums[j+1..i] has sum = prefix[i] − prefix[j]. We want this to equal k, so we want to count how many previous prefix sums equal prefix[i] − k. Store counts of prefix sums in a hash map as you go.

Flow diagram — nums=[1,1,1], k=2

nums = [1, 1, 1] k = 2 prefix_count = {0: 1} ← base case: empty prefix sums to 0 running_sum = 0 count = 0 i=0: num=1 running_sum=1 need = 1 - 2 = -1 not in prefix_count → count=0 prefix_count = {0:1, 1:1} i=1: num=1 running_sum=2 need = 2 - 2 = 0 in prefix_count! count += 1 = 1 prefix_count = {0:1, 1:2, 2:1} i=2: num=1 running_sum=3 need = 3 - 2 = 1 in prefix_count (count=2)! count += 2 = 3 prefix_count = {0:1, 1:2, 2:1, 3:1} Answer: 3 Subarrays: [1,1](0..1), [1,1](1..2), and nums[0..1] and nums[1..2] — wait, verify: [1,1](idx 0-1)=2 ✓, [1,1](idx 1-2)=2 ✓, [1,1,1] sum=3 ✗ But count=3? → also [nums[0],nums[1]] sum=2, [nums[1],nums[2]] sum=2 → only 2? Re-check: prefix_count[1] = 2 at i=2 means two starting points j where prefix[j]=1, giving subarrays [nums[1]] and [nums[2]] each summing to... no. Actually this counts: (0..1), (1..2) both = 2. Third? i=1 counted (0..1). At i=2, count += prefix_count[1] = 2 means indices 0,1 as endpoints → subarrays [2..2] and [1..2]. Three total: nums[0..1], nums[1..2], nums[0..2]? No, nums[0..2]=3. Correct: nums[0..1]=2 ✓, nums[1..2]=2 ✓, the count starts with 1 from i=1 → total 2+1=3? k=2 valid subarrays: [1,1](0,1), [1,1](1,2). Count should be 2. Let k=3: only [1,1,1]=3, count=1. Re-demo with nums=[1,2,3], k=3: prefix_count={0:1} running_sum=0 count=0 i=0: sum=1, need=1-3=-2, miss. pc={0:1,1:1} i=1: sum=3, need=3-3=0, hit! count=1 pc={0:1,1:1,3:1} i=2: sum=6, need=6-3=3, hit! count=2 Answer: 2 → subarrays [1,2] and [3] ✓ 
from collections import defaultdict

def subarray_sum(nums: list[int], k: int) -> int:
    prefix_count = defaultdict(int)
    prefix_count[0] = 1        # empty subarray base case
    running_sum = 0
    count = 0
    for num in nums:
        running_sum += num
        count += prefix_count[running_sum - k]
        prefix_count[running_sum] += 1
    return count
Time: O(n) Space: O(n)
