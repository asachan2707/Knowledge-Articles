Problem 19 · Easy · Pointer Manipulation

Reverse a Linked List

Reverse a singly linked list. Return the new head.

Core insight: you need three pointers simultaneously — prev, curr, and next_node. Save curr.next before overwriting it, flip the pointer, then advance both prev and curr.

Flow diagram — 1 → 2 → 3 → 4 → 5 → None

Initial: prev=None curr=1→2→3→4→5 Step 1: next=2 | 1.next = None | prev=1, curr=2 None ← 1 2 → 3 → 4 → 5 Step 2: next=3 | 2.next = 1 | prev=2, curr=3 None ← 1 ← 2 3 → 4 → 5 Step 3: next=4 | 3.next = 2 | prev=3, curr=4 None ← 1 ← 2 ← 3 4 → 5 Step 4: next=5 | 4.next = 3 | prev=4, curr=5 Step 5: next=None | 5.next = 4 | prev=5, curr=None curr==None → loop ends Return prev (=5): 5 → 4 → 3 → 2 → 1 → None 
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val; self.next = next

# Iterative — O(1) space
def reverse_list(head: ListNode) -> ListNode:
    prev, curr = None, head
    while curr:
        next_node = curr.next
        curr.next = prev
        prev = curr
        curr = next_node
    return prev

# Recursive — O(n) call stack
def reverse_list_rec(head: ListNode) -> ListNode:
    if not head or not head.next:
        return head
    new_head = reverse_list_rec(head.next)
    head.next.next = head
    head.next = None
    return new_head
Time: O(n) Iterative Space: O(1) Recursive Space: O(n) stack
Problem 20 · Easy · Fast & Slow Pointers (Floyd's Algorithm)

Linked List Cycle

Detect if a linked list has a cycle. Do it in O(1) extra space. Bonus: find the start node of the cycle.

Core insight: two runners on a circular track — a fast runner laps a slow one. Move slow by 1 step and fast by 2 steps. If fast reaches None, no cycle. If they meet, there is a cycle. To find the cycle start: reset slow to head and advance both by 1; they meet at the cycle entry point.

Flow diagram — List: 3 → 2 → 0 → -4, -4 → 2 (cycle at 2)

Nodes: 3(0) → 2(1) → 0(2) → -4(3) → [back to 2(1)] Index: 0 1 2 3 Phase 1 — Detect cycle: slow fast Start: 3 3 Step 1: 2 0 (slow+1, fast+2) Step 2: 0 2 (slow+1, fast+2) fast wraps via -4→2 Step 3: -4 -4 ← MEET at -4 Phase 2 — Find cycle start: Reset slow to head (3). Keep fast at meeting point (-4). Advance both by 1 each step: slow: 3→2 fast: -4→2 Meet at 2 ← cycle start Mathematical proof: Let F = distance from head to cycle start Let C = cycle length At meeting point: slow traveled F+a, fast traveled F+a+nC fast = 2×slow → F+a+nC = 2(F+a) → nC = F+a Reset slow to head, advance both by 1: they meet at F steps → cycle start 
def has_cycle(head: ListNode) -> bool:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            return True
    return False

def detect_cycle(head: ListNode) -> ListNode:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast:
            slow = head           # reset to head
            while slow is not fast:
                slow = slow.next
                fast = fast.next
            return slow           # cycle entry
    return None
Time: O(n) Space: O(1)
Problem 21 · Medium · Doubly Linked List + Hash Map

LRU Cache

Design a data structure implementing a Least Recently Used cache. Both get(key) and put(key, value) must run in O(1).

Core insight: combine a hash map (O(1) lookup) with a doubly linked list (O(1) insert/remove). The list maintains access order — most recent at tail, LRU at head. On get or put, move the node to the tail. On eviction, remove from head.

Flow diagram — capacity=2, operations: put(1,1), put(2,2), get(1), put(3,3)

Doubly linked list: dummy_head ↔ ... ↔ dummy_tail (MRU is near tail, LRU is near head) put(1,1): List: [head] ↔ {1:1} ↔ [tail] map={1:node1} put(2,2): List: [head] ↔ {1:1} ↔ {2:2} ↔ [tail] map={1,2} get(1): Move node1 to tail: List: [head] ↔ {2:2} ↔ {1:1} ↔ [tail] Return 1 put(3,3): capacity=2, must evict LRU LRU = node next to head = {2:2} Remove {2:2}: List: [head] ↔ {1:1} ↔ [tail] Add {3:3} at tail: List: [head] ↔ {1:1} ↔ {3:3} ↔ [tail] map={1,3} get(2): key 2 not in map → return -1 
class Node:
    def __init__(self, key=0, val=0):
        self.key = key; self.val = val
        self.prev = self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.cap = capacity
        self.cache = {}                        # key → Node
        self.head = Node()                     # dummy head (LRU side)
        self.tail = Node()                     # dummy tail (MRU side)
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node: Node):
        node.prev.next = node.next
        node.next.prev = node.prev

    def _insert_tail(self, node: Node):
        node.prev = self.tail.prev
        node.next = self.tail
        self.tail.prev.next = node
        self.tail.prev = node

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._remove(node)
        self._insert_tail(node)
        return node.val

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self._remove(self.cache[key])
        node = Node(key, value)
        self.cache[key] = node
        self._insert_tail(node)
        if len(self.cache) > self.cap:
            lru = self.head.next
            self._remove(lru)
            del self.cache[lru.key]
get/put Time: O(1) Space: O(capacity)
Problem 22 · Easy · Stack

Valid Parentheses

Given string s containing only (, ), {, }, [, ], determine if it is valid. A valid string means every open bracket is closed by the same type and in the correct order.

Core insight: use a stack. Push every opening bracket. For every closing bracket, check if the top of the stack is the matching open bracket. If not, or if the stack is empty, return False. At the end, the stack must be empty.

Flow diagram — s="({[]})"

s = "({[]})" stack=[] '(' → opening → push. stack=[(] '{' → opening → push. stack=[(,{] '[' → opening → push. stack=[(,{,[] ']' → closing. top='[', match? Yes → pop. stack=[(,{] '}' → closing. top='{', match? Yes → pop. stack=[(] ')' → closing. top='(', match? Yes → pop. stack=[] Stack empty → True Failure cases: "([)]": '[' on stack but ')' arrives → mismatch → False "([" : loop ends but stack non-empty → False 
def is_valid(s: str) -> bool:
    matching = {')': '(', '}': '{', ']': '['}
    stack = []
    for ch in s:
        if ch in matching:                      # closing bracket
            if not stack or stack[-1] != matching[ch]:
                return False
            stack.pop()
        else:
            stack.append(ch)                    # opening bracket
    return len(stack) == 0
Time: O(n) Space: O(n)

Problem 23 · Medium · Modified Binary Search

Search in Rotated Sorted Array

An integer array was sorted in ascending order, then rotated at some unknown pivot. Find target in O(log n) — return its index or -1 if absent.

Core insight: in a rotated sorted array, one half is always sorted. Check if mid is in the left-sorted or right-sorted portion by comparing nums[mid] with nums[left]. Then check if target falls in the sorted half; if yes, search there, else search the other half.

Flow diagram — nums=[4,5,6,7,0,1,2], target=0

nums = [4, 5, 6, 7, 0, 1, 2] idx: 0 1 2 3 4 5 6 Iter 1: L=0 R=6 mid=3, nums[mid]=7 nums[L]=4 <= nums[mid]=7 → left half [4,5,6,7] is sorted Is target=0 in [4..7]? No (0 < 4) → Search right: L=4 Iter 2: L=4 R=6 mid=5, nums[mid]=1 nums[L]=0 <= nums[mid]=1 → left half [0,1] is sorted Is target=0 in [0..1]? Yes (0 >= 0 and 0 <= 1) → Search left: R=4 Iter 3: L=4 R=4 mid=4, nums[mid]=0 nums[mid] == target → return 4 
def search(nums: list[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        # Determine which half is sorted
        if nums[left] <= nums[mid]:      # left half is sorted
            if nums[left] <= target < nums[mid]:
                right = mid - 1
            else:
                left = mid + 1
        else:                            # right half is sorted
            if nums[mid] < target <= nums[right]:
                left = mid + 1
            else:
                right = mid - 1
    return -1
Time: O(log n) Space: O(1)

  • Find Minimum in Rotated Sorted Array — binary search for the inflection point
  • Search in Rotated Array II (with duplicates) — add nums[left] == nums[mid] edge case: left++
Problem 24 · Hard · Min Heap / Divide & Conquer

Merge K Sorted Lists

Given an array of k linked lists, each sorted in ascending order, merge them all into one sorted linked list and return it.

Core insight: at any moment, the next node in the merged list is the globally minimum head across all k lists. A min heap of size k lets you extract that minimum in O(log k) and push its successor. Total: O(n log k).

Flow diagram — lists=[[1→4→5], [1→3→4], [2→6]]

Heap (min-heap by node value, stores (val, idx, node)): Init heap: [(1,0,L0), (1,1,L1), (2,2,L2)] Extract (1,L0): output→1, push L0.next=(4): heap=[(1,L1),(2,L2),(4,L0)] Extract (1,L1): output→1, push L1.next=(3): heap=[(2,L2),(3,L1),(4,L0)] Extract (2,L2): output→2, push L2.next=(6): heap=[(3,L1),(4,L0),(6,L2)] Extract (3,L1): output→3, push L1.next=(4): heap=[(4,L0),(4,L1),(6,L2)] Extract (4,L0): output→4, push L0.next=(5): heap=[(4,L1),(5,L0),(6,L2)] Extract (4,L1): output→4, L1.next=None → don't push: heap=[(5,L0),(6,L2)] Extract (5,L0): output→5, L0.next=None → don't push: heap=[(6,L2)] Extract (6,L2): output→6, L2.next=None → heap empty Result: 1→1→2→3→4→4→5→6 
import heapq

def merge_k_lists(lists: list[ListNode]) -> ListNode:
    # heap stores (val, tie_break_idx, node)
    heap = []
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    dummy = ListNode()
    curr = dummy
    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node
        curr = curr.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
Time: O(n log k) — n total nodes, k lists Space: O(k) — heap size at most k

Alternative: Divide & Conquer

Pair up lists and merge each pair. Repeat on the resulting smaller set of lists. Log k rounds, each round O(n) total → same O(n log k) time but avoids heap overhead. 

def merge_two(l1: ListNode, l2: ListNode) -> ListNode:
    dummy = ListNode()
    curr = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next = l1; l1 = l1.next
        else:
            curr.next = l2; l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2
    return dummy.next

def merge_k_divide(lists: list[ListNode]) -> ListNode:
    if not lists:
        return None
    while len(lists) > 1:
        merged = []
        for i in range(0, len(lists), 2):
            l1 = lists[i]
            l2 = lists[i+1] if i+1 < len(lists) else None
            merged.append(merge_two(l1, l2))
        lists = merged
    return lists[0]
