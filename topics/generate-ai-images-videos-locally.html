Generate Unlimited AI Images and Videos Locally — No Internet, No Limits

Cloud image APIs charge per generation and impose content filters. Running Stable Diffusion, ComfyUI, and open video models locally gives you unlimited generation at the cost of GPU memory. This guide covers the full stack: models, UIs, hardware requirements, and a JavaScript API to drive it all programmatically.

ToolsComfyUI · Automatic1111 · Ollama · Wan2.1 · AnimateDiff
LevelIntermediate — assumes basic terminal comfort
UpdatedJune 2026

The shift to local generation is not about avoiding cost alone — it is about removing the middleman. Local models respond in seconds (on capable hardware), never refuse a creative prompt for policy reasons, and produce consistent style because you control the exact model checkpoint and sampler settings every time.

Core idea: Local AI generation = model + UI/API + hardware. Get these three right and you have an unlimited, private creative engine that runs entirely on your machine.

Hardware requirements

What GPU you actually need

Minimum

8 GB VRAM

RTX 3070, RTX 4060Ti, M2/M3 Mac (unified). SD 1.5 and SDXL in 512px–768px. ~4–10s/image. No video at full quality.

Recommended

16–24 GB VRAM

RTX 3090, RTX 4090, M3 Max/M4 Pro. SDXL 1024px, FLUX.1, short video clips (5s @ 480p). ~1–3s/image.

Power user

48–80 GB VRAM

RTX 6000 Ada, A100, H100, dual 4090, M4 Ultra. FLUX.1 Pro, full HD video, ControlNet + LoRA stacking. Sub-1s/image.

Apple Silicon note: unified memory counts as VRAM. An M3 Max with 48 GB handles most tasks that would require an RTX 4090.

Image generation stack

The three tools and when to use each

1. Automatic1111 (stable-diffusion-webui)

The original web UI — massive extension ecosystem, beginner-friendly, excellent for checkpoints and LoRAs.

# Launch (Mac)
./webui.sh --api --listen
# Launch (Windows/Linux)
./webui.sh --api --listen --xformers

2. ComfyUI — node-based workflow engine

Preferred for automation. Every operation is a node you can wire together and export as a JSON workflow, then drive via API.

git clone https://github.com/comfyanonymous/ComfyUI
cd ComfyUI
pip install -r requirements.txt
# Put models in models/checkpoints/
python main.py --listen 0.0.0.0 --port 8188

3. FLUX.1 — state-of-the-art 2026 model

FLUX.1 [dev] and [schnell] (distilled, 4-step) outperform SDXL on photorealism and prompt adherence.

# Via Ollama (easiest path for FLUX schnell)
ollama pull flux1-schnell
ollama run flux1-schnell "A cyberpunk city at rain, neon reflections, 8k"

# Or load the raw weights in ComfyUI's FLUX workflow
Video generation

Open video models in 2026

Wan2.1 — best open-source video model

Released by Alibaba in 2025, Wan2.1 produces 5-second 720p clips from text or image prompts. Runs on 16 GB VRAM with the quantised 14B variant. 

# Install via ComfyUI-WanVideoWrapper node
# Or use the direct Python API:
pip install wan2.1

from wan.generate import VideoGenerator

gen = VideoGenerator(model="wan2.1-14b-q4", device="cuda")
video = gen.text_to_video(
  prompt="A golden eagle soaring over mountain peaks at sunset, cinematic, slow motion",
  num_frames=49,       # ~4.9s at 10fps
  width=1280,
  height=720,
  guidance_scale=7.5,
)
video.save("eagle.mp4")

AnimateDiff — animate existing SD models

Converts any SD 1.5 checkpoint into a video model using a motion LoRA adapter. Short clips (8–32 frames), excellent for stylised animation.

CogVideoX — longer structured video

6B and 13B open models from Zhipu AI. Best for structured scene progression and longer clips (up to 10s).

JavaScript API integration

Driving ComfyUI from Node.js

ComfyUI exposes a REST + WebSocket API. This pattern queues a workflow, polls for completion, and downloads the image: 

// comfyui-client.js
const COMFY_URL = 'http://localhost:8188';

async function generateImage({ prompt, negative = '', steps = 20, cfg = 7 }) {
  // Minimal workflow object (SD 1.5 text2img)
  const workflow = {
    "3": { class_type: "KSampler",         inputs: { seed: Math.floor(Math.random() * 1e15), steps, cfg, sampler_name: "euler", scheduler: "normal", denoise: 1, model: ["4", 0], positive: ["6", 0], negative: ["7", 0], latent_image: ["5", 0] } },
    "4": { class_type: "CheckpointLoaderSimple", inputs: { ckpt_name: "dreamshaper_8.safetensors" } },
    "5": { class_type: "EmptyLatentImage",  inputs: { width: 512, height: 512, batch_size: 1 } },
    "6": { class_type: "CLIPTextEncode",    inputs: { text: prompt,   clip: ["4", 1] } },
    "7": { class_type: "CLIPTextEncode",    inputs: { text: negative, clip: ["4", 1] } },
    "8": { class_type: "VAEDecode",         inputs: { samples: ["3", 0], vae: ["4", 2] } },
    "9": { class_type: "SaveImage",         inputs: { filename_prefix: "output", images: ["8", 0] } },
  };

  const clientId = crypto.randomUUID();

  // Queue the prompt
  const { prompt_id } = await fetch(`${COMFY_URL}/prompt`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt: workflow, client_id: clientId }),
  }).then(r => r.json());

  // Poll history until done
  while (true) {
    await new Promise(r => setTimeout(r, 500));
    const history = await fetch(`${COMFY_URL}/history/${prompt_id}`).then(r => r.json());
    if (history[prompt_id]) {
      const outputs = history[prompt_id].outputs;
      const images  = Object.values(outputs).flatMap(n => n.images ?? []);
      // Download the first image
      const img = images[0];
      const blob = await fetch(
        `${COMFY_URL}/view?filename=${img.filename}&subfolder=${img.subfolder}&type=${img.type}`
      ).then(r => r.arrayBuffer());
      return Buffer.from(blob);
    }
  }
}

// Usage
const imageBuffer = await generateImage({ prompt: "astronaut on mars, epic lighting, 4k" });
require('fs').writeFileSync('output.png', imageBuffer);
console.log('Image saved!');

Automatic1111 REST API

// a1111-client.js
async function txt2img({ prompt, negative_prompt = '', width = 512, height = 512, steps = 20 }) {
  const res = await fetch('http://localhost:7860/sdapi/v1/txt2img', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ prompt, negative_prompt, width, height, steps, cfg_scale: 7, sampler_name: 'Euler a' }),
  });
  const { images } = await res.json();
  // images[0] is base64-encoded PNG
  return Buffer.from(images[0], 'base64');
}

const buf = await txt2img({ prompt: "a cozy coffee shop, rainy window, warm light, impressionist" });
require('fs').writeFileSync('coffee.png', buf);
Models and checkpoints

Where to find models

  • Civitai.com — largest checkpoint and LoRA repository. Community-rated. Download via API or browser.
  • HuggingFace Hub — official model releases (FLUX.1, SDXL, CogVideoX, Wan2.1).
  • Ollama registry — easiest install path for FLUX schnell and LLaVA-based vision models.
  • SafeTensors format — always prefer .safetensors over .ckpt — safer, faster to load.

Key model families

  • SD 1.5 — small (2 GB), fast, huge LoRA ecosystem, slightly dated quality.
  • SDXL 1.0 — 6.5 GB, 1024px native, much better composition and text rendering.
  • FLUX.1 [schnell] — 4-step distilled, 12 GB, best prompt adherence as of 2026.
  • FLUX.1 [dev] — full model, slower but richer detail, requires 24 GB VRAM at full precision.
  • Wan2.1-14B-Q4 — quantised video model, 16 GB VRAM, 720p clips.

Key Takeaway

Local AI image and video generation has crossed the usability threshold. FLUX.1 and Wan2.1 match or exceed cloud-API quality on capable hardware, ComfyUI's JSON workflow API lets you integrate generation into any Node.js pipeline, and the cost per image after hardware amortises rounds to zero. The barrier is VRAM, not skill.
