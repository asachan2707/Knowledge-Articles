I Wasted 6 Months Using Claude Code Wrong. Here Are the 14 Commands That Changed Everything.

Most Claude Code users type natural language and wait. That is using 20% of the tool. The other 80% is a command system, keyboard shortcuts, and session management features that most users never discover — because they are not in the obvious places.

Sourcecode.claude.com/docs — CLI reference, interactive-mode, best-practices
LevelIntermediate — you have used Claude Code, but not the commands

Claude Code is an agentic coding environment, not a chat interface. The distinction matters because it has a completely different set of capabilities hidden behind a simple text input. Most people never find them. Here are the 14 that change how you work.

The first thing to learn: Shift+Tab cycles through permission modes (default → acceptEdits → plan → auto). Plan mode is read-only — Claude explores and proposes without touching a file. Use it before any non-trivial task.

Session foundation

1. /init — instant project intelligence

Analyses your codebase and auto-generates a CLAUDE.md with build commands, test runners, and conventions it discovers. Most users spend weeks re-explaining project context every session. Run /init once and Claude carries that context forward.

2. /plan — enter read-only mode before touching code

Switches Claude to plan mode. Claude reads files, explores the codebase, and proposes a plan before making any edits. The most common Claude Code mistake is jumping to execution and getting the wrong implementation. /plan costs 2 minutes and saves hours. Keyboard shortcut: Shift+Tab.

3. /clear — the most under-used command

Resets conversation context while keeping project memory (CLAUDE.md and auto-memory persist). Use between unrelated tasks. If you have corrected Claude twice on the same mistake in one session, that is a sign — /clear and rewrite a better prompt rather than accumulating corrections in the context.

4. /compact [instructions] — targeted summarisation

Unlike /clear, /compact summarises the conversation to free context while maintaining thread continuity. Pass instructions to control what is preserved: /compact Focus on the API changes from the last 20 messages. Essential for long working sessions on complex features.

Navigation and recovery

5. Esc (single) — interrupt without losing work

Stops Claude mid-action while preserving the full context. Most users let a bad direction run to completion, accumulating context pollution and having to backtrack. One Esc redirects immediately. Use it the moment Claude starts heading in the wrong direction.

6. Esc + Esc (or /rewind) — time travel

Opens the rewind menu to restore conversation state, code state, or both to any previous checkpoint. Every file edit is auto-snapshotted. This is not git — it is finer-grained, and it survives session close. Made a bad decision three changes ago? Rewind.

7. @filename — direct file injection

Type @ anywhere in your prompt to trigger file path autocomplete and include the file's full content without waiting for Claude to discover it. @ is faster and exact; describing where code lives is slower and often ambiguous.

8. ! command — shell mode

Prefix any input with ! to run a shell command and add both the command and its output to context. ! npm test, ! git status, ! curl -s localhost:3000/health. Exit with Escape. Asking Claude to run commands conversationally is slower.

Context and awareness

9. /context [all] — see what is eating your window

Visualises context usage as a grid with per-item token costs and optimisation suggestions. Most users have no idea why Claude "forgets" instructions mid-session. /context usually reveals an MCP server, a large file, or a stale command output consuming hundreds of tokens silently.

10. /btw <question> — side question without context pollution

Asks a clarifying question in an overlay. The answer never enters conversation history. Works even while Claude is mid-task. Asking "quick questions" as regular messages bloats context with exchanges that are not relevant to the main task.

Automation and scale

11. /goal [condition] — autonomous execution with a gate

Sets a goal condition Claude works toward across turns until met. /goal all tests pass. Claude evaluates the condition after every turn and keeps working. Most users babysit Claude turn-by-turn; /goal hands off the loop.

12. claude --bg "task" — background agent

Starts a session as a background agent and returns your terminal immediately. Monitor with claude agents. Use /background mid-session to detach. The most underused feature for long-running tasks — run it while you do other work.

13. /batch <instruction> — parallel codebase-wide changes

Researches the codebase, decomposes work into 5–30 independent units, and runs each in its own isolated git worktree as a background sub-agent. Each produces its own PR. For large migrations, refactors, or codebase-wide changes that would be impossible to do in a single linear session.

14. /code-review [--fix] [--comment] — automated review with fixes

Reviews the current diff for correctness bugs in a fresh context. --fix applies findings directly. --comment posts them as inline GitHub PR comments. ultra level runs multi-agent cloud review for major changes. Asking "review my code" as a chat message shares context with your current session — this command runs in isolation, which produces better results.

Keyboard shortcuts to memorise

The non-obvious ones

  • Ctrl+G — open current prompt in your default text editor (for long prompts)
  • Ctrl+O — toggle transcript viewer (shows all tool calls, expands collapsed MCP calls)
  • Option+P — switch model without clearing your current prompt
  • Option+T — toggle extended thinking mode
  • Ctrl+R — searchable command history (by project or all-time)
  • Shift+Tab — cycle permission modes
  • Ctrl+T — toggle task list
The three big mistakes

What most users get wrong

  • The kitchen-sink session: mixing unrelated tasks in one long session. Each task degrades the next. Fix: /clear between tasks — it takes two seconds.
  • The bloated CLAUDE.md: over 200 lines causes Claude to lose track of which rules matter. Fix: prune aggressively; delete anything Claude already does correctly without being told.
  • No verification criteria: asking Claude to implement something without giving it a test, a build command, or a comparison screenshot. Fix: always give Claude a check it can run autonomously to validate its own work.

Key Takeaway

Claude Code's value is not the chat interface — it is the command system, the session management, and the automation features layered underneath. Learn /plan, /clear, /compact, @-file reference, and /goal first. Those five alone will change your daily workflow more than any amount of prompt tuning.
