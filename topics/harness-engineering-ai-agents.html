Harness Engineering: What Makes AI Agents Perform Better Than 80% of Others

The harness is everything around the model call: how you structure tasks, constrain output, manage state, handle failures, verify results, and chain agents together. Most AI agents underperform not because the model is weak but because the harness is thin. This guide shows exactly how to build one that is robust.

LevelIntermediate–Advanced
FocusProduction agent design patterns with JavaScript examples

A harness is the scaffolding you build around an LLM call to make it reliable, observable, and composable. A raw LLM call is a suggestion engine. A well-engineered harness is a software system that happens to use an LLM internally — with typed contracts, error handling, retry logic, verification steps, and audit trails.

Core idea: The model is only one component. The harness — task decomposition, output schema enforcement, verification agents, failure recovery, and observability — determines 90% of a production agent's reliability.

What separates good from bad harnesses

Two teams build a document analysis agent. Team A prompts the model, gets text back, parses it with regex, and ships. Team B defines a typed output schema, validates every response, runs a verification agent on high-stakes outputs, retries with richer context on schema violations, logs every call with cost and latency, and tests the harness with an eval suite. Three weeks in, Team A's agent is failing on 15% of documents and nobody knows why. Team B's agent has a 99.2% success rate and they know exactly where the other 0.8% fails.

The eight pillars

What a harness must do

🎯

1. Task decomposition

Break complex goals into atomic sub-tasks that each have a clear output contract.

📋

2. Schema enforcement

Force structured output via tool use. Never parse free text with regex when a schema will do.

🔁

3. Retry with context

When schema validation fails, retry with the error message included so the model self-corrects.

✅

4. Verification agents

Spawn a second agent to verify high-stakes outputs. Adversarial checking beats hoping.

🔒

5. Risk gates

Classify every tool call by impact. Gate writes and external actions behind confirmation or approval flows.

📊

6. Observability

Log every call: prompt, response, model, tokens, cost, latency, success/failure. Trace multi-step chains.

🧪

7. Evals

Write tests for the harness, not just the model. Test retry logic, schema fallbacks, and verification paths.

⚡

8. Parallelism

Identify independent sub-tasks and run them in parallel. Sequential agents for parallel work waste wall-clock time.

Pillar 1 & 2: Decomposition + Schema

Typed, decomposed task pipeline

import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

// Define a strict output schema for each atomic task
const EXTRACT_SCHEMA = {
  name: 'extract_requirements',
  description: 'Extract structured requirements from a document',
  input_schema: {
    type: 'object',
    properties: {
      functional: {
        type: 'array',
        items: { type: 'object', properties: {
          id:       { type: 'string' },
          title:    { type: 'string' },
          priority: { type: 'string', enum: ['must', 'should', 'could'] },
        }, required: ['id', 'title', 'priority'] }
      },
      constraints: { type: 'array', items: { type: 'string' } },
      ambiguities: { type: 'array', items: { type: 'string' } },
    },
    required: ['functional', 'constraints', 'ambiguities'],
  },
};

async function extractRequirements(document) {
  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 2048,
    tools: [EXTRACT_SCHEMA],
    tool_choice: { type: 'tool', name: 'extract_requirements' },
    messages: [{ role: 'user', content: `Extract requirements from:\n\n${document}` }],
  });

  const toolUse = response.content.find(b => b.type === 'tool_use');
  return toolUse.input; // Typed, validated object
}
Pillar 3: Retry with context

Self-correcting retry loop

// Retry harness — feeds schema errors back to the model
async function callWithRetry(prompt, schema, maxRetries = 3) {
  const messages = [{ role: 'user', content: prompt }];
  let lastError = null;

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    // On retry, inject the previous error into context
    if (lastError) {
      messages.push(
        { role: 'assistant', content: [{ type: 'text', text: 'I need to correct my previous response.' }] },
        { role: 'user',      content: `Your previous response had this validation error: ${lastError}. Please fix it and return valid output.` }
      );
    }

    const response = await client.messages.create({
      model: 'claude-sonnet-4-6',
      max_tokens: 2048,
      tools: [schema],
      tool_choice: { type: 'tool', name: schema.name },
      messages,
    });

    const toolUse = response.content.find(b => b.type === 'tool_use');
    if (!toolUse) { lastError = 'No tool call returned'; continue; }

    const validation = validateSchema(toolUse.input, schema.input_schema);
    if (validation.valid) return toolUse.input;

    lastError = validation.errors.join('; ');
    console.warn(`Attempt ${attempt} failed: ${lastError}`);
  }

  throw new Error(`Schema validation failed after ${maxRetries} attempts: ${lastError}`);
}

// Simple schema validator
function validateSchema(data, schema) {
  const errors = [];
  for (const field of schema.required ?? []) {
    if (data[field] === undefined) errors.push(`Missing required field: ${field}`);
  }
  return { valid: errors.length === 0, errors };
}
Pillar 4: Verification agents

Adversarial verification for high-stakes output

// Two-agent pipeline: generator + adversarial verifier
async function generateAndVerify(prompt) {
  // Step 1: Primary agent generates output
  const primary = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 2048,
    messages: [{ role: 'user', content: prompt }],
  });
  const draft = primary.content[0].text;

  // Step 2: Independent verifier tries to find flaws
  const verifier = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 1024,
    tools: [{
      name: 'verdict',
      description: 'Return verification verdict',
      input_schema: {
        type: 'object',
        properties: {
          approved: { type: 'boolean' },
          issues:   { type: 'array', items: { type: 'string' } },
          confidence: { type: 'number', minimum: 0, maximum: 1 },
        },
        required: ['approved', 'issues', 'confidence'],
      },
    }],
    tool_choice: { type: 'tool', name: 'verdict' },
    messages: [{
      role: 'user',
      content: `You are an adversarial reviewer. Your job is to FIND PROBLEMS in this output.
Be skeptical. If anything looks wrong, flag it.\n\nOriginal task: ${prompt}\n\nOutput to verify:\n${draft}`,
    }],
  });

  const { approved, issues, confidence } = verifier.content.find(b => b.type === 'tool_use').input;

  if (!approved || confidence < 0.8) {
    console.warn('Verification failed:', issues);
    // Could retry, escalate, or return with warnings
    return { output: draft, verified: false, issues };
  }

  return { output: draft, verified: true, issues: [] };
}
Pillar 6: Observability

Tracing every call

// Observability wrapper around every LLM call
class TracedClient {
  #client;
  #traces = [];

  constructor(client) { this.#client = client; }

  async create(params, label = 'unnamed') {
    const start = performance.now();
    let response, error;

    try {
      response = await this.#client.messages.create(params);
    } catch (e) {
      error = e;
      throw e;
    } finally {
      const duration = performance.now() - start;
      const trace = {
        label,
        model:     params.model,
        timestamp: new Date().toISOString(),
        duration:  Math.round(duration),
        tokens:    response?.usage,
        cost:      this.#estimateCost(params.model, response?.usage),
        success:   !error,
        error:     error?.message,
      };
      this.#traces.push(trace);
      console.log(`[TRACE] ${label}: ${trace.duration}ms, ${trace.tokens?.output_tokens ?? 0} out tokens, $${trace.cost?.toFixed(4)}`);
    }

    return response;
  }

  #estimateCost(model, usage) {
    if (!usage) return 0;
    const rates = {
      'claude-sonnet-4-6': { input: 3/1e6, output: 15/1e6 },
      'claude-haiku-4-5-20251001': { input: 0.25/1e6, output: 1.25/1e6 },
    };
    const r = rates[model] ?? { input: 3/1e6, output: 15/1e6 };
    return usage.input_tokens * r.input + usage.output_tokens * r.output;
  }

  getTraces() { return this.#traces; }
  getTotalCost() { return this.#traces.reduce((s, t) => s + (t.cost ?? 0), 0); }
}

// Usage
const traced = new TracedClient(client);
const res = await traced.create({ model: 'claude-sonnet-4-6', max_tokens: 512, messages: [...] }, 'extract-requirements');
console.log('Session cost: $' + traced.getTotalCost().toFixed(4));
Pillar 8: Parallelism

Parallel task execution

// Run independent analysis tasks in parallel
async function analyseDocument(doc) {
  // These tasks don't depend on each other — run all at once
  const [requirements, risks, timeline] = await Promise.all([
    callWithRetry(
      `Extract functional requirements from:\n${doc}`,
      REQUIREMENTS_SCHEMA
    ),
    callWithRetry(
      `Identify technical risks in:\n${doc}`,
      RISKS_SCHEMA
    ),
    callWithRetry(
      `Estimate implementation timeline for:\n${doc}`,
      TIMELINE_SCHEMA
    ),
  ]);

  // Synthesis step — depends on all three
  const synthesis = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 1024,
    messages: [{
      role: 'user',
      content: `Synthesise into a project brief:
Requirements: ${JSON.stringify(requirements)}
Risks: ${JSON.stringify(risks)}
Timeline: ${JSON.stringify(timeline)}`,
    }],
  });

  return {
    requirements,
    risks,
    timeline,
    brief: synthesis.content[0].text,
  };
}
// Wall-clock time = max(requirements, risks, timeline) time, not sum.
Putting it together

The complete harness pattern

class AgentHarness {
  #traced;

  constructor(apiKey) {
    const client = new Anthropic({ apiKey });
    this.#traced  = new TracedClient(client);
  }

  async run(task, schema, options = {}) {
    const { maxRetries = 3, verify = false, parallel = false } = options;

    if (parallel && Array.isArray(task)) {
      return Promise.all(task.map(t => this.run(t, schema, { maxRetries, verify })));
    }

    const output = await callWithRetry(task, schema, maxRetries);

    if (verify) {
      const { verified, issues } = await generateAndVerify(task);
      if (!verified) console.warn('Verification issues:', issues);
      return { output, verified, issues };
    }

    return { output, verified: null };
  }

  getStats() {
    return {
      traces:    this.#traced.getTraces().length,
      totalCost: this.#traced.getTotalCost(),
    };
  }
}

// Usage
const harness = new AgentHarness(process.env.ANTHROPIC_API_KEY);
const result  = await harness.run(
  'Analyse this requirements doc and extract structured data',
  EXTRACT_SCHEMA,
  { maxRetries: 3, verify: true }
);
console.log('Stats:', harness.getStats());

Key Takeaway

Harness engineering is the discipline that separates demo agents from production agents. Eight pillars — task decomposition, schema enforcement, retry with context, verification agents, risk gates, observability, evals, and parallelism — together create a system that is reliable enough to trust with real work. The model is interchangeable. The harness is your competitive moat.
