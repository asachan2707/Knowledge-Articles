Anthropic's Engineer Said "Kill Markdown." Here's What He Actually Meant.

The viral Anthropic engineering post told Claude Code users to strip Markdown from their prompts. The internet interpreted this as "never use Markdown in any AI context." That reading is wrong — and expensive if you apply it uniformly.

What he actually said: Claude Code renders its responses in a terminal. Terminals don't render Markdown. So **bold** appears literally as asterisks, ## Heading as a hash symbol, and bullet points as hyphens. The instruction was: "when your Claude Code output goes to a terminal, disable Markdown formatting in responses."

The correct mental model: format for the renderer, not for the model. If the output renders in a web UI or document editor → Markdown is fine and often better. If the output goes to a terminal, a plain-text file, or a TTS pipeline → turn off Markdown.

// In Claude Code: tell Claude to skip Markdown in terminal responses
// Add to CLAUDE.md or system prompt:
// "Format responses as plain text. No markdown, no bullet asterisks,
//  no hash headings. Terminal output only."

// In your app: control format via system prompt based on output target
function buildSystemPrompt(outputTarget) {
  const formatInstruction = outputTarget === 'terminal' || outputTarget === 'plain'
    ? 'Use plain text. No markdown formatting.'
    : 'Use clear markdown formatting with headers and bullet lists where helpful.';

  return `You are a helpful AI assistant. ${formatInstruction}`;
}

// For streaming to a web chat UI: Markdown is great
const webPrompt   = buildSystemPrompt('web');

// For piping output to a shell script: plain text
const cliPrompt   = buildSystemPrompt('terminal');

Rule: Markdown in AI responses is a presentation decision, not a quality decision. Match the format to the renderer. "Kill Markdown" means "don't render formatting codes as literal characters in a terminal."

A Single CLAUDE.md File Went Viral. The Reason Is Embarrassingly Simple.

A developer posted their CLAUDE.md file on X/Twitter. It got 40,000 impressions in 24 hours. The file was 31 lines. People were stunned not by the complexity — but by how much Claude's behaviour changed with just 31 lines of clear instruction.

The viral file contained exactly four things:

  1. The tech stack (three lines)
  2. Three hard constraints: "tests must pass before committing," "no console.log in prod," "always use the logger at src/lib/logger.ts"
  3. Two style rules: "two spaces, single quotes"
  4. One pointer: "see DECISIONS.md for architectural choices"
# MyApp

> Node.js REST API. PostgreSQL. Redis. Deployed on Fly.io.

## Stack
- Runtime: Node.js 22 / TypeScript 5 strict
- DB: PostgreSQL via Drizzle ORM (queries in src/db/)
- Cache: Redis (sessions + BullMQ job queue)

## Non-negotiables
- Run `npm test` before any commit. STOP if tests fail.
- No console.log in production code — use src/lib/logger.ts
- All secrets via process.env — never hardcoded

## Style
- 2-space indent, single quotes, no trailing semicolons
- Named exports only

## Architecture context
- See DECISIONS.md for technical decisions
- See openapi.yaml for the API contract

The embarrassingly simple lesson: Claude's default behaviour is already good. You don't need 300 lines to get great results. You need a handful of precise, non-negotiable rules that override the defaults that are wrong for your specific project. The viral CLAUDE.md worked because it was small enough to actually be read and followed completely.

Rule: A 30-line CLAUDE.md that Claude follows 100% is worth more than a 300-line one Claude only half-follows. Start with your three most important non-negotiables and your stack. Nothing else is required to get most of the value.

5 Books That Made Me Smarter Than 99% of People

Not "smarter" in IQ — smarter in the sense of having mental models that consistently produce better decisions. These five books changed how the author thinks in categories, not just added information to a pile.

  • Thinking, Fast and Slow — Daniel Kahneman System 1 (fast, intuitive, error-prone) vs System 2 (slow, deliberate, effortful). Reading this makes you permanently better at spotting when your fast brain is lying to you. Most financial, social, and technical decisions are made with System 1. Knowing when to force System 2 is the core skill of experts.
  • Designing Data-Intensive Applications — Martin Kleppmann The only technical book on this list. It does not teach you technologies — it teaches you to think about data, consistency, durability, and distributed systems the way great engineers do. The mental models from this book will still be useful in 20 years when all the technologies have changed.
  • The Personal MBA — Josh Kaufman A compressed mental model set for understanding business. Not an MBA substitute — a primer that explains value creation, pricing, systems, and human psychology in 400 pages. Useful for engineers who want to understand why business decisions are made.
  • Atomic Habits — James Clear The systems > goals mental model. Outcomes are lagging indicators of systems. Most people set goals (outcomes) and neglect the system. This book explains how to design the system in terms of environment design, habit stacking, and friction reduction — not motivation.
  • The Almanack of Naval Ravikant — Eric Jorgenson Free PDF. Wealth and happiness through specific knowledge, leverage, and long-term thinking. Naval's mental models on compounding (skills, money, relationships), the difference between judgment and effort, and how to think about career optionality are concise and high-density.

Pattern: The books that compound intelligence share a structure — they give you a new lens rather than new facts. A lens changes how you see every situation in a domain forever. Facts are forgotten; lenses are permanent.

Health · Nutrition · Productivity · 3 min

This ONE Nutrient Everyone Should Include In Their Diet

The nutrient is Omega-3 fatty acids — specifically EPA and DHA (the long-chain forms found in fatty fish and algae oil). Not because of cardiovascular benefits (though those are real) — but because of what they do to cognitive function, inflammation, and mood in knowledge workers.

Why it matters for knowledge workers specifically:

  • Brain composition: 60% of the brain's dry weight is fat. DHA is the primary structural fat in neurons. Deficiency is associated with reduced memory, slower processing speed, and increased cognitive decline risk.
  • Inflammation and focus: EPA has anti-inflammatory effects in the brain. Chronic low-grade brain inflammation (from poor diet, poor sleep, chronic stress) is associated with brain fog, reduced executive function, and depression. EPA directly counters this.
  • Mood regulation: Multiple meta-analyses show EPA supplementation reduces depressive symptoms at doses above 1g EPA/day. The mechanism is partially serotonin and dopamine system support.
  • Most people are deficient: Western diets have an Omega-6 to Omega-3 ratio of ~15:1 to 20:1. The research-supported target is 4:1 or less. The gap is not subtle.

Practical action:

  • Food sources: fatty fish (salmon, sardines, mackerel) 2–3 times per week provides adequate EPA/DHA for most people.
  • Supplement if needed: 1–2g EPA+DHA per day from fish oil or algae oil (vegan). Look for third-party tested products (IFOS certification).
  • Algae oil for vegans: fish get their Omega-3s from algae. Algae oil is the original source and avoids heavy metal concerns in large fish.
  • Timing: with meals containing fat — Omega-3s are fat-soluble and absorb better with dietary fat.

Rule: No supplement replaces sleep, exercise, and whole food. But among micronutrients, Omega-3 has the strongest evidence base for cognitive function in healthy adults and is the most commonly deficient in tech workers' diets. Start with fatty fish twice a week before buying anything.

Key Takeaways

  • Kill Markdown: format for the renderer, not the model. Plain text for terminals, Markdown for web UIs.
  • Viral CLAUDE.md: 31 precise lines beat 300 vague ones. Three non-negotiables plus your stack is enough to unlock most of the value.
  • 5 Books: read for lenses, not facts. Kahneman, Kleppmann, Kaufman, Clear, Ravikant.
  • One Nutrient: Omega-3 (EPA + DHA). Fatty fish 2–3x/week or 1–2g supplement. Strongest evidence base for cognitive function among micronutrients.
