Setup

Installing and running with Ollama

# Install Ollama (macOS)
brew install ollama
# Or: curl -fsSL https://ollama.com/install.sh | sh  (Linux)

# Pull and run the best coding model for your RAM tier
ollama pull qwen2.5-coder:32b-instruct-q4_K_M   # 24 GB+ RAM
ollama pull qwen2.5-coder:14b-instruct-q4_K_M   # 12–16 GB RAM
ollama pull phi4:14b-q4_K_M                      # 8–12 GB RAM

# Start the Ollama server (runs on http://localhost:11434)
ollama serve

# Quick test
ollama run qwen2.5-coder:32b-instruct-q4_K_M \
  "Write a Node.js function that debounces async functions with proper cancellation"
JavaScript integration

Calling local Ollama from Node.js

// ollama-client.js — OpenAI-compatible API
// Ollama exposes an OpenAI-compatible endpoint at /v1

import OpenAI from 'openai'; // works with Ollama!

const ollama = new OpenAI({
  baseURL: 'http://localhost:11434/v1',
  apiKey:  'ollama', // Ollama ignores the key but the SDK requires one
});

// 1. Basic code completion
async function complete(prompt, model = 'qwen2.5-coder:32b-instruct-q4_K_M') {
  const response = await ollama.chat.completions.create({
    model,
    messages: [
      { role: 'system', content: 'You are an expert JavaScript engineer. Return only code, no explanation.' },
      { role: 'user',   content: prompt },
    ],
    temperature: 0.1, // low temp for code — determinism matters
  });
  return response.choices[0].message.content;
}

// 2. Streaming completion (great for VS Code extension / CLI)
async function completeStreaming(prompt, onChunk) {
  const stream = await ollama.chat.completions.create({
    model: 'qwen2.5-coder:14b-instruct-q4_K_M',
    messages: [{ role: 'user', content: prompt }],
    stream: true,
  });

  for await (const chunk of stream) {
    const text = chunk.choices[0]?.delta?.content ?? '';
    if (text) onChunk(text);
  }
}

// 3. Code review helper
async function reviewCode(filePath, code) {
  return complete(`Review this ${filePath} file for bugs, security issues, and improvements:\n\n${code}`);
}

// 4. Batch refactor — send multiple files
async function refactorFiles(files) {
  const results = await Promise.all(
    files.map(({ name, content }) =>
      complete(`Refactor this ${name} to use modern async/await and fix any issues:\n\n${content}`)
        .then(refactored => ({ name, refactored }))
    )
  );
  return results;
}

// Usage
const code = await complete('Write a rate limiter using a token bucket algorithm in Node.js');
console.log(code);

Using Ollama's native API directly

// Direct Ollama API (without OpenAI SDK)
async function ollamaChat(messages, model = 'qwen2.5-coder:14b-instruct-q4_K_M') {
  const res = await fetch('http://localhost:11434/api/chat', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ model, messages, stream: false }),
  });
  const data = await res.json();
  return data.message.content;
}

// List available models
async function listModels() {
  const res = await fetch('http://localhost:11434/api/tags');
  const { models } = await res.json();
  return models.map(m => ({ name: m.name, size: m.size, modified: m.modified_at }));
}

// Check if a model is loaded (warm)
async function isModelLoaded(model) {
  const res  = await fetch('http://localhost:11434/api/ps');
  const data = await res.json();
  return data.models?.some(m => m.name === model) ?? false;
}

Quantisation guide

Choosing the right quantisation for your RAM

  • Q8_0 — near full-precision quality, ~1.1× model size in RAM. Use if you have headroom.
  • Q4_K_M — best quality-per-GB trade-off. Default recommendation. ~4× compression vs fp16.
  • Q4_K_S — slightly smaller than K_M, slightly lower quality. Use when K_M won't fit.
  • Q3_K_M — noticeable quality drop on hard reasoning. Only if K_M doesn't fit.
  • Q2_K — significant degradation. Avoid for coding tasks.

Rule: Q4_K_M is the default. Only go lower if the model won't fit. Going higher rarely pays off enough to justify the RAM cost.

When to use local vs cloud

Decision framework

  • Use local when: working with private code, need offline access, running many completions (cost control), want low latency on M-series hardware.
  • Use cloud when: problem requires state-of-the-art reasoning (Claude Opus/Sonnet, GPT-4o), need multi-modal (vision), require very long context (>128k tokens), prototyping speed matters more than privacy.
  • Hybrid setup: local model for autocomplete and quick edits; cloud model for architecture review, complex debugging, and multi-file refactors.
