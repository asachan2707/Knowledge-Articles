LLMs.txt — The Hidden File Changing How AI Reads the Internet

Like robots.txt guided search crawlers, llms.txt is a plain-text convention that tells AI language models what your site is about, which pages matter, and how to summarise your content accurately. Adopted quietly in 2025–2026, it is becoming essential for any site that wants to be represented fairly in AI-generated answers.

StandardProposed by Answer.AI / Jeremy Howard, 2025
LevelBeginner — no prior knowledge needed
StatusWidely adopted, not yet IETF formal

Every major website has a robots.txt file that search engines have respected for 30 years. AI crawlers are different — they do not just index, they summarise, and the summaries end up inside LLM answers. llms.txt is the convention that gives you control over that process.

Core idea: llms.txt is a Markdown file at the root of your domain that gives AI models a curated, machine-readable overview of your site — so they paraphrase you accurately instead of guessing from fragments.

Why this exists

A documentation site for a popular library found that ChatGPT and Claude were confidently giving users outdated API signatures — pulled from cached snippets, not the current docs. The maintainers added an llms.txt pointing to the canonical, versioned reference pages. Within weeks AI answers started citing the right version. The problem was not hallucination in the classic sense; it was bad source selection. llms.txt fixes the source.

The format

What goes in llms.txt

The file lives at https://yourdomain.com/llms.txt and follows a simple Markdown schema:

# YourSiteName

> One-sentence description of what this site is and who it is for.

Optional longer paragraph explaining context, product focus, or scope.

## Core Docs
- [Getting Started](https://yourdomain.com/docs/start): First steps, installation, hello world
- [API Reference](https://yourdomain.com/docs/api): Full parameter reference

## Guides
- [Deployment Guide](https://yourdomain.com/guides/deploy): Production deployment checklist
- [Troubleshooting](https://yourdomain.com/guides/debug): Common errors and fixes

## Optional
- [Changelog](https://yourdomain.com/changelog): Version history
- [Blog](https://yourdomain.com/blog): Release announcements

llms-full.txt — the deep variant

Some sites also serve llms-full.txt, which embeds the actual Markdown content of key pages inline rather than just linking to them. This trades file size for self-containment — an AI that ingests llms-full.txt does not need to follow links.

Sub-topics covered

  • Root placement: must be at /llms.txt, not a subdirectory.
  • H1 = site name: the blockquote after it becomes the one-line description.
  • Sections with H2: group links by purpose — Core Docs, Guides, API, Changelog, etc.
  • Link annotations: text after the colon is the per-link description visible to the model.
  • Optional section: links here are deprioritised — useful for blogs, social, marketing pages.
  • No crawl directives here: use robots.txt for that — llms.txt is about context, not access.
JavaScript example

Generating llms.txt dynamically in Node.js

If your docs are code-generated or versioned, automate llms.txt generation as part of your build pipeline: 

// generate-llms-txt.js
import fs from 'fs';
import path from 'path';

const SITE = 'https://docs.mylib.dev';

// Pull your nav structure from wherever it lives
const sections = [
  {
    heading: 'Core Docs',
    pages: [
      { url: '/docs/start',    title: 'Getting Started',   desc: 'Install and run hello world in 5 minutes' },
      { url: '/docs/api',      title: 'API Reference',     desc: 'Full typed parameter reference, v3.x' },
      { url: '/docs/config',   title: 'Configuration',     desc: 'All config keys with defaults' },
    ],
  },
  {
    heading: 'Guides',
    pages: [
      { url: '/guides/deploy', title: 'Deployment',        desc: 'Production checklist and env vars' },
      { url: '/guides/perf',   title: 'Performance',       desc: 'Benchmarks and tuning tips' },
    ],
  },
  {
    heading: 'Optional',
    pages: [
      { url: '/changelog',     title: 'Changelog',         desc: 'Version history' },
    ],
  },
];

function buildLlmsTxt(siteName, description, sections) {
  const lines = [
    `# ${siteName}`,
    '',
    `> ${description}`,
    '',
  ];

  for (const section of sections) {
    lines.push(`## ${section.heading}`);
    for (const page of section.pages) {
      lines.push(`- [${page.title}](${SITE}${page.url}): ${page.desc}`);
    }
    lines.push('');
  }

  return lines.join('\n').trimEnd() + '\n';
}

const output = buildLlmsTxt(
  'MyLib Docs',
  'JavaScript utility library for server-side data processing. Supports Node 18+.',
  sections
);

fs.writeFileSync(path.join('public', 'llms.txt'), output, 'utf8');
console.log('llms.txt written:', output.length, 'bytes');

Serving it from an Express app

import express from 'express';
import fs from 'fs';

const app = express();

// Serve llms.txt with correct content-type (plain text)
app.get('/llms.txt', (req, res) => {
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');
  res.setHeader('Cache-Control', 'public, max-age=86400'); // 1 day
  res.send(fs.readFileSync('./public/llms.txt', 'utf8'));
});

// llms-full.txt — embed page content inline for offline-capable AI ingestion
app.get('/llms-full.txt', async (req, res) => {
  const pages = await Promise.all([
    fetch('https://docs.mylib.dev/docs/start.md').then(r => r.text()),
    fetch('https://docs.mylib.dev/docs/api.md').then(r => r.text()),
  ]);
  const full = pages.join('\n\n---\n\n');
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');
  res.send(full);
});
Who reads it and how

AI systems that consume llms.txt

  • Claude with web search: Anthropic's retrieval layer checks for llms.txt to select authoritative pages before fetching raw HTML.
  • ChatGPT browsing / Operator agents: OpenAI agents use it for site-level context when exploring a domain.
  • Perplexity: uses llms.txt in its indexer to weight which pages get summarised for answers.
  • RAG pipelines: custom retrieval systems can seed their vector stores from llms.txt link lists rather than full sitemaps.
  • MCP tools / web-search agents: any agent that navigates the web can read llms.txt first to reduce crawl depth.

Relationship to robots.txt and sitemaps

These three files do different things and all three should coexist:

  • robots.txt — access control: "you may / may not crawl this path."
  • sitemap.xml — completeness: "here is every URL on this site."
  • llms.txt — curation: "here are the pages that matter most and what they say."
Adoption and tooling

Current state in 2026

  • llmstxt.org — the reference spec and a directory of sites that have adopted it.
  • Next.js plugin — next-llms-txt auto-generates from your next.config.js nav config.
  • Docusaurus plugin — official plugin outputs llms.txt from sidebar config on build.
  • Astro integration — @astrojs/llms-txt generates both variants during astro build.
  • Cloudflare Workers — serve dynamically from edge with 0ms cache TTL.
  • Adoption rate: as of mid-2026, ~40% of developer-facing documentation sites include it; general consumer sites lag behind.

Key Takeaway

llms.txt is the smallest change with the largest impact on how AI represents your content. It costs ten minutes to create, reduces AI hallucination about your product, and is increasingly expected by the AI tools your users rely on. If you publish documentation, ship it today.
